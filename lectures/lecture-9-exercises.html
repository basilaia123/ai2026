🤖 ლექცია 8 - სავარჯიშოები

ავტომატიზაცია და No-Code ინსტრუმენტები

📚 სავარჯიშოების მიზანი

ამ სავარჯიშოების მიზანია პრაქტიკული გამოცდილების მიღება ავტომატიზაციისა და No-Code ინსტრუმენტების გამოყენებაში. თქვენ შექმნით რეალურ workflow-ებს, მოახდენთ რუტინული ამოცანების ავტომატიზაციას და გამოსცდით პოპულარულ პლატფორმებს.

🔰 დამწყები დონის სავარჯიშოები

სავარჯიშო 1: Zapier-ით პირველი ავტომატიზაცია
მარტივი ⏱️ 15-20 წთ

🎯 მიზანი

შექმენით მარტივი Zap, რომელიც ავტომატურად აგზავნის შეტყობინებას, როდესაც ახალ იმეილს იღებთ.

🛠️ საჭირო ინსტრუმენტები

  • Zapier უფასო ანგარიში (zapier.com)
  • Gmail ანგარიში
  • Slack ან Discord ანგარიში

📋 ნაბიჯები:

  1. Zapier-ში დარეგისტრირდით: გადადით zapier.com-ზე და შექმენით უფასო ანგარიში
  2. შექმენით ახალი Zap: დააწკაპუნეთ "Create Zap" ღილაკზე
  3. Trigger Setup:
    • აირჩიეთ Gmail როგორც Trigger App
    • აირჩიეთ Event: "New Email"
    • დააკავშირეთ თქვენი Gmail ანგარიში
    • ფილტრის დაყენება: წერილები მხოლოდ კონკრეტული გამომგზავნისაგან
  4. Action Setup:
    • აირჩიეთ Slack ან Discord როგორც Action App
    • აირჩიეთ Event: "Send Channel Message"
    • დააკავშირეთ თქვენი Slack/Discord
    • აირჩიეთ არხი და შექმენით შეტყობინების შაბლონი
  5. Test და Activate:
    • გატესტეთ Zap სატესტო იმეილით
    • დარწმუნდით, რომ შეტყობინება მივიდა
    • ჩართეთ Zap
💡 რჩევა: გამოიყენეთ Gmail-ის ფილტრები, რათა გაფილტროთ მხოლოდ მნიშვნელოვანი იმეილები (მაგალითად: შეკვეთები, alert-ები). ეს თავიდან აგაცილებთ სპამ შეტყობინებებს.

🌟 ბონუს დავალება:

დაამატეთ მეორე Action - შეინახეთ email-ის დეტალები Google Sheets-ში. ეს დაგეხმარებათ თვალყურის დევნებაში.

სავარჯიშო 2: Make.com-ით მარტივი Workflow
მარტივი ⏱️ 20-25 წთ

🎯 მიზანი

შექმენით Make.com-ის სცენარი, რომელიც RSS არხიდან ახალ სტატიებს Twitter-ზე გამოაქვეყნებს.

🛠️ საჭირო ინსტრუმენტები

  • Make.com უფასო ანგარიში (make.com)
  • Twitter/X ანგარიში
  • ნებისმიერი RSS feed (მაგალითად: Medium, Blog)

📋 ნაბიჯები:

  1. Make.com-ში შესვლა: დარეგისტრირდით make.com-ზე
  2. ახალი Scenario: Create a new scenario
  3. RSS Module:
    • დაამატეთ "RSS" > "Watch RSS feed items"
    • ჩასვით RSS feed URL (მაგალითად: https://medium.com/feed/@username)
    • დააყენეთ: Check every 15 minutes
  4. Twitter Module:
    • დაამატეთ "Twitter" > "Create a Tweet"
    • დააკავშირეთ Twitter ანგარიში
    • შექმენით tweet template: "{{title}} - {{link}}"
  5. Test და Schedule:
    • Run once to test
    • გადაამოწმეთ Twitter
    • ჩართეთ scheduling
💡 რჩევა: დაამატეთ Text parser, რომ გააუმჯობესოთ ტვიტის ტექსტი - შეამოკლოთ სათაური 200 სიმბოლომდე და დაამატოთ ჰეშთეგები.
სავარჯიშო 3: Microsoft Power Automate - Email Automation
მარტივი ⏱️ 15-20 წთ

🎯 მიზანი

ავტომატურად შეინახეთ იმეილის დანართები OneDrive-ში და მიიღეთ შეტყობინება Teams-ში.

🛠️ საჭირო ინსტრუმენტები

  • Microsoft 365 ანგარიში
  • OneDrive
  • Microsoft Teams (ოფციონალური)

📋 ნაბიჯები:

  1. Power Automate-ში შესვლა: გადადით flow.microsoft.com
  2. Create Flow: "Automated cloud flow"
  3. Trigger:
    • "When a new email arrives" (Outlook)
    • სათაურის ფილტრი: "Invoice" ან "Report"
    • აქვს დანართი: დიახ
  4. Action 1 - Save Attachment:
    • "OneDrive - Create file"
    • საქაღალდე: /Documents/Invoices/
    • ფაილის სახელი: {{Subject}} - {{DateReceived}}
    • ფაილის შიგთავსი: {{Attachments Content}}
  5. Action 2 - Notify (ოფციონალური):
    • "Teams - Post message"
    • Message: "ახალი დოკუმენტი შენახულია: {{Subject}}"
  6. Test: გაგზავნეთ სატესტო იმეილი attachment-ით
💡 რჩევა: დაამატეთ Condition - შეამოწმეთ დანართი ტიპი (.pdf, .xlsx) და სხვადასხვა ფოლდერში შეინახეთ.

🔶 საშუალო დონის სავარჯიშოები

სავარჯიშო 4: Multi-Step Workflow - Google Forms to Notion
საშუალო ⏱️ 30-40 წთ

🎯 მიზანი

შექმენით workflow, რომელიც Google Forms-იდან მიღებულ პასუხებს ავტომატურად ამატებს Notion-ის მონაცემთა ბაზაში, აგზავნის დადასტურების იმეილს და ქმნის Slack-ის შეტყობინებას.

🛠️ საჭირო ინსტრუმენტები

  • Google Forms
  • Notion (უფასო ანგარიში)
  • Zapier ან Make.com
  • Gmail
  • Slack

📋 ნაბიჯები:

  1. შექმენით Google Form:
    • სახელი, Email, კომპანია, მესიჯი
    • დააკავშირეთ Google Sheets-თან
  2. Notion Database Setup:
    • შექმენით მონაცემთა ბაზა: "Contact Requests"
    • თვისებები: Name, Email, Company, Message, Status, Date
  3. Zapier/Make Workflow:
    • Trigger: Google Forms - New Response
    • Action 1: Notion - Create Database Item
    • Action 2: Gmail - Send Email (confirmation)
    • Action 3: Slack - Send Message (team notification)
  4. Email Template: 
    Subject: Thanks for contacting us, {{Name}}!

Hi {{Name}},

Thank you for reaching out! We've received your message
about {{Message}}.

Our team will review your request and get back to you
within 24 hours.

Best regards,
Your Team
  5. Test: შეავსეთ Form და გადაამოწმეთ ყველა ნაბიჯი
💡 რჩევა: დაამატეთ ფილტრი - თუ email-ში არის "urgent", გაგზავნეთ SMS ან განსხვავებული notification პრიორიტეტით.

🌟 ბონუს დავალება:

დაამატეთ ChatGPT API ინტეგრაცია - ავტომატურად გააანალიზეთ შეტყობინების სენტიმენტი და დაამატეთ თეგი Notion-ში (Positive/Neutral/Negative).

სავარჯიშო 5: Social Media Auto-Posting Schedule
საშუალო ⏱️ 35-45 წთ

🎯 მიზანი

შექმენით სოციალური მედიის პოსტების ავტომატური დაგეგმვის სისტემა, რომელიც Google Sheets-ში შეტანილ პოსტებს ავტომატურად გამოაქვეყნებს Twitter-ზე, LinkedIn-სა და Facebook-ზე.

🛠️ საჭირო ინსტრუმენტები

  • Google Sheets
  • Make.com ან Zapier
  • Social Media ანგარიშები (Twitter, LinkedIn, Facebook)
  • ChatGPT API (ოფციონალური - ჰეშთეგების გენერაცია)

📋 ნაბიჯები:

  1. Google Sheets Template:
    • სვეტები: Date, Time, Content, Platform, Status, Image_URL, Hashtags
    • შეავსეთ 5-10 პოსტით მომავალი კვირისთვის
  2. Make.com Scenario:
    • დაგეგმვის ტრიგერი: Every 30 minutes
    • Google Sheets: Search rows where:
      • Date = Today
      • Time <= Current Time + 30 min
      • Status = "Pending"
    • როუტერის მოდული: განშტოება პლატფორმის მიხედვით
    • Twitter-ის შტო: Create Tweet (if Platform = Twitter)
    • LinkedIn-ის შტო: Share Update (if Platform = LinkedIn)
    • Facebook-ის შტო: Create Page Post (if Platform = Facebook)
    • ცხრილის განახლება: Change Status to "Posted"
  3. Test:
    • დააყენეთ სატესტო პოსტი 5 წუთში
    • გადაამოწმეთ ყველა პლატფორმა
💡 რჩევა: გამოიყენეთ ChatGPT API, რომ ავტომატურად დაამატოთ რელევანტური ჰეშთეგები თითოეულ post-ს content-ის საფუძველზე.
⚠️ გაფრთხილება: API-ის გამოყენების ლიმიტების გამო, არ დააყენოთ ძალიან ხშირი შემოწმება (min 15-30 წთ).
სავარჯიშო 6: E-commerce Order Processing Automation
საშუალო ⏱️ 40-50 წთ

🎯 მიზანი

შექმენით შეკვეთის დამუშავების სრული სამუშაო პროცესი - ახალი შეკვეთისას: შეინახეთ მონაცემები, გაგზავნეთ დადასტურება, შეატყობინეთ გუნდს და შექმენით ინვოისი.

🛠️ საჭირო ინსტრუმენტები

  • Google Forms (შეკვეთის იმიტირებული ფორმა)
  • Airtable ან Google Sheets
  • Gmail
  • Make.com ან Zapier
  • Slack/Discord

📋 ნაბიჯები:

  1. შეკვეთის ფორმა:
    • Product, Quantity, Customer Name, Email, Address, Phone
  2. სამუშაო პროცესის ნაბიჯები:
    • Trigger: New Form Response
    • Step 1: შეკვეთის ID-ის გენერირება (TIMESTAMP + RANDOM)
    • Step 2: Airtable/Sheets-ში შენახვა with Status: "New"
    • Step 3: ჯამის გამოთვლა (Quantity × Price)
    • Step 4: დადასტურების იმეილის გაგზავნა with Order ID
    • Step 5: გუნდის შეტყობინება Slack-ში with order details
    • Step 6: PDF ინვოისის შექმნა (using PDFMonkey or similar)
    • Step 7: ინვოისის გაგზავნა იმეილით
  3. იმეილის შაბლონები: 
    Subject: Order Confirmation #{{OrderID}}

Dear {{CustomerName}},

Thank you for your order!

Order Details:
- Product: {{Product}}
- Quantity: {{Quantity}}
- Total: {{Total}} GEL
- Order ID: {{OrderID}}

Your order will be processed within 24 hours.

Attached: Invoice

🌟 ბონუს დავალება:

დაამატეთ შემდგომი კონტროლის სისტემა: 24 საათის შემდეგ ავტომატურად გაგზავნეთ "Order Status Update" email. გამოიყენეთ დაყოვნების მოდული.

🔴 მაღალი დონის სავარჯიშოები

სავარჯიშო 7: AI-ზე დაფუძნებული კონტენტის შექმნის ჯაჭვი
რთული ⏱️ 60-75 წთ

🎯 მიზანი

შექმენით სრული, AI-ზე დაფუძნებული კონტენტის შექმნის პროცესი: Notion-იდან თემების იდეების აღება → ChatGPT-ის მიერ კონტენტის გენერირება → Grammarly-ით შემოწმება → WordPress-ის მონახაზში შენახვა → გუნდის შეტყობინება.

🛠️ საჭირო ინსტრუმენტები

  • Notion
  • OpenAI API (ChatGPT)
  • Make.com Advanced
  • WordPress ან Medium
  • Slack

📋 ნაბიჯები:

  1. Notion Database Setup:
    • მონაცემთა ბაზა: "Content Ideas"
    • Properties: Topic, Status, Keywords, Target_Length, Generated_Content, Publishing_Date
    • შეავსეთ 5 თემის იდეა Status: "Ready for Generation"
  2. Make.com Advanced Scenario:
    • Trigger: Notion - Watch Database Items (filter: Status = "Ready for Generation")
    • Step 1: ChatGPT API გამოძახება 
      პრომპტის ნიმუში:
"Write a comprehensive blog post about {{Topic}}.
Include these keywords: {{Keywords}}.
Target length: {{Target_Length}} words.
Style: Professional and engaging.
Include: Introduction, 3 main points, conclusion."
    • Step 2: ტექსტის დამუშავება - Clean and format
    • Step 3: WordPress - მონახაზი პოსტის შექმნა
      • Title: {{Topic}}
      • Content: {{Generated_Content}}
      • Status: Draft
      • Categories, Tags from Keywords
    • Step 4: Notion-ის განახლება
      • Status: "Generated - Awaiting Review"
      • Generated_Content: Save content
      • WordPress_Link: Draft link
    • Step 5: Slack-ის შეტყობინება 
      New draft ready for review! 📝

Topic: {{Topic}}
Length: {{Word_Count}} words
Draft: {{WordPress_Link}}

Please review and approve for publishing.
  3. შეცდომების დამუშავება:
    • დაამატეთ Error Handler - თუ API-ის შეცდომა, send error notification
    • ხელახალი ცდის ლოგიკა: 3 attempts
💡 რჩევა: გამოიყენეთ GPT-4 უკეთესი ხარისხისთვის. დაამატეთ ტემპერატურის პარამეტრი (0.7-0.9) კრეატიულობისთვის.

🌟 ბონუს დავალება:

  1. დაამატეთ DALL-E-სთან ინტეგრაცია - ავტომატურად შექმენით მთავარი სურათი topic-ის საფუძველზე
  2. დაამატეთ SEO ოპტიმიზაცია - ChatGPT-ს სთხოვეთ მეტა აღწერა და საკვანძო სიტყვების გენერაცია
  3. გამოქვეყნების დაგეგმვა - დაამატეთ დადასტურების ღილაკი Slack-ში → ავტომატურად გამოაქვეყნეთ WordPress-ზე
⚠️ ხარჯები: OpenAI API არის ფასიანი ($0.002/1K tokens for GPT-3.5). გაითვალისწინეთ ბიუჯეტი.
სავარჯიშო 8: მომხმარებელთა მხარდაჭერის ავტომატიზაციის სისტემა
რთული ⏱️ 75-90 წთ

🎯 მიზანი

შექმენით მომხმარებელთა მხარდაჭერის სრული ავტომატიზაცია: შემოსული იმეილები (ბილეთები) → AI კლასიფიკაცია → მარშრუტიზაცია → ავტომატური პასუხი → CRM-ის განახლება → ესკალაციის სისტემა.

🛠️ საჭირო ინსტრუმენტები

  • Gmail ან Outlook
  • OpenAI API
  • Airtable (CRM მონაცემთა ბაზა)
  • Make.com Advanced
  • Slack

📋 ნაბიჯები:

  1. Airtable CRM Setup:
    • ცხრილი: "Support Tickets"
    • ველები: Ticket_ID, Customer_Email, Subject, Category, Priority, Status, Assigned_To, Created_Date, Response
  2. სამუშაო პროცესის არქიტექტურა:
    • Trigger: New Email to support@yourdomain.com
    • Step 1: ბილეთის ID-ის გენერირება
    • Step 2: AI კლასიფიკაცია (ChatGPT API) 
      Prompt:
"Classify this customer email into category and priority:

Email: {{Email_Body}}

Categories: Technical, Billing, General, Feature Request
Priority: Low, Medium, High, Urgent

Return JSON:
{
  "category": "...",
  "priority": "...",
  "suggested_response": "..."
}"
    • Step 3: როუტერი კატეგორიის მიხედვით
      • Technical → Route to Tech Team
      • Billing → Route to Finance
      • General → Auto-respond
      • Feature Request → Save to Product Board
    • Step 4: Airtable CRM-ში შენახვა
    • Step 5: ავტომატური პასუხის გაგზავნა 
      Subject: Re: {{Original_Subject}} [Ticket: {{Ticket_ID}}]

Dear {{Customer_Name}},

Thank you for contacting us. We've received your {{Category}}
request and assigned ticket ID: {{Ticket_ID}}.

Priority: {{Priority}}
Expected Response Time: {{SLA_Time}}

{{AI_Suggested_Response}}

Our team will get back to you soon.

Best regards,
Support Team
    • Step 6: გუნდის შეტყობინება (Slack)
      • Post in relevant channel (#tech-support, #billing, etc.)
      • Include: Priority, Category, Ticket Link
    • Step 7: ესკალაციის ლოგიკა
      • IF Priority = "Urgent" → Send SMS to Manager
      • IF Priority = "High" → Direct Slack DM
  3. შემდგომი კონტროლის ავტომატიზაცია:
    • After 24h - check if Status still "Open"
    • If yes - send reminder to assigned person
    • After 72h - escalate to manager

🌟 ბონუს დავალება:

  1. AI პასუხის გენერირება: ChatGPT-თან full response generation FAQs-ის საფუძველზე
  2. სენტიმენტ ანალიზი: გააანალიზეთ customer sentiment და დაამატეთ flag
  3. ცოდნის ბაზასთან ინტეგრაცია: ავტომატურად მოძებნეთ relevant KB articles და ჩართეთ response-ში
  4. მრავალენოვანი მხარდაჭერა: Auto-detect language → respond in same language
