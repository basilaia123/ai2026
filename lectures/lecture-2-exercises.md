# ლექცია 2: Prompt Engineering - პრაქტიკული ვარჯიშები

## 🎯 ვარჯიშების მიზნები

ამ პრაქტიკული ვარჯიშების შესრულების შემდეგ სტუდენტები შეძლებენ:
- **შექმნან ეფექტური prompt-ები** სხვადასხვა სცენარისთვის
- **გამოიყენონ role-based prompting** ტექნიკა
- **დაუფლდნენ few-shot learning** მეთოდს
- **გამოიყენონ chain of thought** რთულ ამოცანებში
- **მართონ ფორმატი და ტონი** სხვადასხვა აუდიტორიისთვის
- **იდენტიფიცირებენ და გამოასწორონ** ცუდ prompt-ებს

## 📚 ვარჯიში 1: Prompt-ის ანატომია (20 წუთი)

#### ნაბიჯი 1: ზოგადი Prompt-ის ანალიზი (5 წუთი)

- [ ] როლი არ არის განსაზღვრული?
- [ ] ფორმატი არ არის მითითებული?
- [ ] ტონი არ არის დაზუსტებული?

## 🎓 ვარჯიში 3: Few-Shot Learning პრაქტიკა (30 წუთი)

### 🎯 მიზანი:
Few-shot learning ტექნიკის გამოყენებით კონკრეტული output-ის მიღება

### 📝 ვარჯიშის ნაკრები:

#### სცენარი 1: პროდუქტის Review (10 წუთი)

**დავალება:** შექმენით few-shot prompt რომელიც გააგენერებს review-ებს კონკრეტული ფორმატით.

**თქვენი Few-Shot Prompt:**
```
შექმენი პროდუქტის review შემდეგი ფორმატით:

მაგალითი 1:
პროდუქტი: [პროდუქტი 1]
ვარსკვლავები: ⭐⭐⭐⭐⭐
რას მომწონს: [3 პუნქტი]
რას არ მომწონს: [1-2 პუნქტი]
რეკომენდაცია: [ვისთვის შეესაბამება]

მაგალითი 2:
პროდუქტი: [პროდუქტი 2]
ვარსკვლავები: ⭐⭐⭐⭐
რას მომწონს: [3 პუნქტი]
რას არ მომწონს: [1-2 პუნქტი]
რეკომენდაცია: [ვისთვის შეესაბამება]

ახლა შექმენი review:
პროდუქტი: [თქვენი არჩეული პროდუქტი]
```

**ჩაწერეთ თქვენი სრული prompt აქ:**


**დავალება:** შექმენით few-shot prompt სხვადასხვა ტიპის ბიზნეს email-ებისთვის.

```
[2 წინადადება - კონტექსტი]
[შეხვედრის მოთხოვნა და მიზანი]
[შემოთავაზებული დრო და ადგილი]

[თქვენი მაგალითი]

[თქვენი მაგალითი]

ახლა შექმენი:
```
