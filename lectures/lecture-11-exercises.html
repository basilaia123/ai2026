🤖 ლექცია 10 - სავარჯიშოები

Custom AI Assistants - პრაქტიკული დავალებები

📋 მიმოხილვა

ეს სავარჯიშოები დაგეხმარებათ, მიიღოთ პრაქტიკული გამოცდილება მორგებული AI ასისტენტების შექმნასა და კონფიგურაციაში. თითოეული სავარჯიშო აერთიანებს თეორიულ ცოდნას რეალურ პროექტებთან.

სავარჯიშოების რაოდენობა: 5 ძირითადი + 2 დამატებითი

სავარჯიშოების დრო: 2-3 საათი (ძირითადი)

⚠️ მნიშვნელოვანი

  • ყველა სავარჯიშოს შესასრულებლად დაგჭირდებათ ChatGPT Plus ან Enterprise ანგარიში (Custom GPTs-ისთვის)
  • ან Claude.ai Pro ანგარიში (Projects-ისთვის)
  • შეინახეთ ყველა დოკუმენტაცია და თქვენი სამუშაოს ეკრანის სურათები (screenshots)
  • ტესტირება ჩაატარეთ რეალურ სიტუაციებში

📝 სავარჯიშო 1: პირადი ასისტენტის შექმნა

⏱️ დრო: 30-40 წუთი

  2. შექმენით Custom GPT შემდეგი პარამეტრებით:
    • სახელი და აღწერა (დაწერეთ ქართულად და ინგლისურად)

    • საუბრის დამწყები ფრაზები - მინიმუმ 4 მაგალითი
    • ცოდნის ბაზა - ატვირთეთ 2-3 დოკუმენტი თქვენი სფეროდან
    • შესაძლებლობები - აირჩიეთ შესაბამისი ფუნქციები
  3. ტესტირება:
    • ჩაატარეთ მინიმუმ 5 სხვადასხვა ტიპის საუბარი
    • ჩაიწერეთ რა მუშაობს კარგად და რა საჭიროებს გაუმჯობესებას
    • გადაიღეთ ეკრანის სურათები

მაგალითი ინსტრუქციები (მარკეტინგის მენეჯერისთვის):

You are a Marketing Campaign Assistant specialized in digital marketing for the Georgian market. Your role is to:

  • Help create marketing campaigns tailored for a Georgian audience
  • Analyze campaign performance and provide actionable insights
  • Generate creative content ideas considering the cultural context
  • Provide best practices for social media platforms popular in Georgia
  • Always ask clarifying questions about the target audience, budget, and goals
  • Use a data-driven approach when analyzing results
  • Speak in Georgian when discussing with the client, use English for technical terms

📤 შედეგები (ჩასაბარებელი მასალები):

  • Custom GPT-ის სრული კონფიგურაცია (ეკრანის სურათი)
  • ინსტრუქციების სრული ტექსტი

  • გაუმჯობესების რეკომენდაციები (150-200 სიტყვა)

მაგალითი Project: კურსის შექმნა

Project Name: "AI Course Development Assistant"

Custom Instructions:

You are an instructional design assistant for creating an AI/ML course.

Context:
- Target audience: Professionals wanting to integrate AI in their work
- Course duration: 11 sessions (22 hours)
- Language: Georgian with English technical terms
- Delivery: In-person with online materials

Your responsibilities:
1. Help develop lecture content following course syllabus
2. Create practical exercises aligned with learning objectives
3. Design assessments and homework assignments
4. Ensure content progression and proper pacing
5. Maintain consistent formatting and style

Standards to follow:
- Each lecture: 2 hours (120 minutes)
- Include: slides, summary, exercises, homework
- Use FiraGO font for Georgian text
- Follow color scheme: #667eea (primary), #764ba2 (secondary)
- All files: UTF-8 encoding

When creating content:

📝 დამატებითი სავარჯიშო C: AI Agent Design (Conceptual)

⏱️ დრო: 40-50 წუთი

🎯 მიზანი

გაიგოთ AI Agents-ის კონცეპცია და დაგეგმოთ agent architecture რეალური use case-ისთვის (ბონუს სავარჯიშო - კოდირება არ არის საჭირო).

⚠️ Note:

ეს არის conceptual/planning exercise - კოდის წერა არ არის საჭირო! მიზანია გაიგოთ როგორ მუშაობს agents და როდის არის მათი გამოყენება გამართლებული.

📋 ამოცანა

სცენარი: თქვენ ხართ Smart Academy-ს ტექნიკური დირექტორი. გინდათ შექმნათ AI Agent რომელიც ავტომატურად შეაგროვებს და დაამუშავებს student feedback-ს ლექციების შემდეგ.

Ნაბიჯი 1: Agent Goal & Requirements (10 წთ)

დაწერეთ დეტალური აღწერა:

  1. Main Goal: რა უნდა მიაღწიოს agent-მა?
  2. Input: რა მონაცემები აქვს ხელმისაწვდომი?
    • Student feedback forms (Google Forms responses)?
    • Email-ების feedback-ით?
    • ჩატის ისტორია?
  3. Output: რა უნდა შექმნას?
    • Summary report?
    • Action items for improvement?
    • Alert თუ critical issues-ია?
  4. Success Criteria: როდის ითვლება წარმატებულად?

Ნაბიჯი 2: Agent Architecture Design (15 წთ)

დაგეგმეთ 5 კომპონენტი:

1. 🎯 Goal/Task

მაგალითი: "Collect feedback from Google Forms → Analyze sentiment → Identify top issues → Create weekly report → Alert instructor if critical issues"

2. 🧠 Reasoning Engine

რომელი LLM გამოიყენებთ? GPT-4? Claude? რატომ?

3. 🛠️ Tools/Actions

რა tools დასჭირდება agent-ს?

  • Google Forms API (read responses)?
  • Gmail API (send reports)?
  • Sentiment analysis tool?
  • Database (store historical data)?
  • Data visualization (create charts)?

ჩამოთვალეთ ყველა!

4. 💾 Memory

რა უნდა გახსოვდეს agent-ს?

  • Short-term: მიმდინარე feedback batch?
  • Long-term: ისტორიული ტრენდები? previous reports?
5. 🔄 Action Loop

დაწერეთ step-by-step workflow:

  1. Observe: Check Google Forms for new responses
  2. Think: Any new feedback since last check?
  3. Act: Fetch new responses
  4. Think: Is analysis needed? (more than 10 responses?)
  5. Act: Run sentiment analysis...
  6. ...

Ნაბიჯი 3: Risk Assessment (10 წთ)

რა რისკები არსებობს ამ agent-ისთვის?

  1. Cost: რამდენს დახარჯავს თვეში? (estimate LLM calls)
    • თუ 100 feedback თვეში → X agent cycles → Y LLM calls → Z ღირებულება?
  2. Privacy: student feedback შეიძლება იყოს sensitive!
    • როგორ დაიცავთ პირადულობას?
    • ანონიმიზაცია საჭიროა?
  3. Errors: რა მოხდება თუ agent-მა არასწორად გაანალიზა feedback?
    • False negative (გამოტოვა critical issue)?
    • False positive (არასწორად დაადგინა პრობლემა)?
  4. Security: API access რისკები?

Ნაბიჯი 4: Alternative Solution (5 წთ)

Question: ამ use case-სთვის ნამდვილად საჭიროა AI Agent?

ალტერნატივები:

  • Custom GPT რომ ინსტრუქტორმა manually upload-ს feedback და მიიღოს summary?
  • Simple automation (Zapier) + ChatGPT API call?
  • Manual review ყოველ კვირას?

შეადარეთ:

Solution Cost Complexity Autonomy რჩევა
Full AI Agent $50-100/mo High Full ?
Custom GPT + Manual $20/mo Low None ?
Zapier + API $30/mo Medium Partial ?

რომელს აირჩევდით? რატომ?

📤 შედეგები:

  • Agent Goal & Requirements დოკუმენტი (200-300 სიტყვა)
  • 5-component Architecture დიზაინი (დეტალური)
  • Step-by-step Action Loop (10-15 steps)
  • Risk Assessment ანალიზი (ყველა 4 რისკზე)
  • Alternative Solutions შედარება + რეკომენდაცია
  • Final Recommendation: Agent თუ Alternative? რატომ? (100-150 სიტყვა)

✅ რას ისწავლით ამ სავარჯიშოდან:

  • როგორ იფიქროთ agent architecture-ზე systematically
  • როგორ შეაფასოთ cost vs benefit agent-ისთვის
  • როდის არის agent-ი overkill და ალტერნატივა უკეთესია
  • რა რისკები არსებობს და როგორ მივაგოთ მათ
  • რეალური use case-ის დაგეგმვის პრაქტიკა

💡 Key Insight: უმეტეს შემთხვევაში, Custom GPT + Manual/Semi-automated workflow საკმარისია. Agents საჭიროა მხოლოდ მაშინ როცა full autonomy აუცილებელია!
