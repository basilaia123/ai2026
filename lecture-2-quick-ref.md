## 📐 Prompt-ის 6 ელემენტი (RITCEF Framework)

```
┌─────────────────────────────────────────────┐
│  R  Role       - როლი და პერსპექტივა      │
│  I  Instruction- მთავარი ინსტრუქცია        │
│  T  Task       - კონკრეტული დავალება       │
│  C  Context    - საჭირო კონტექსტი          │
│  E  Examples   - მაგალითები (few-shot)      │
│  F  Format     - სასურველი ფორმატი         │
└─────────────────────────────────────────────┘
```

### ⚡ სწრაფი ტემპლეიტი:
```
შენ ხარ [Role].
[Context] კონტექსტში,
შეასრულე [Task],
მიჰყევი [Instruction],
გამოიყენე [Format] ფორმატი.

მაგალითები:
[Example 1]
[Example 2]
```

## 🎯 Role-Based Prompting - სწრაფი როლები

### 💼 პროფესიული როლები

#### ბიზნეს ანალიტიკოსი:
```
შენ ხარ გამოცდილი ბიზნეს ანალიტიკოსი. შეაანალიზე [მონაცემები/სიტუაცია],
იდენტიფიცირე trends, რისკები და შესაძლებლობები.
წარმოადგინე SWOT ანალიზის ფორმატით.
```

#### კონტენტ სტრატეგი:
```
როგორც კონტენტ სტრატეგი, შემოგთავაზე 5 სათაური [თემა]-ზე.
თითოეული უნდა იყოს: SEO-optimized, engaging და click-worthy.
დაურთე თითოეულს meta description (150 სიმბოლო).
```

#### Tech Lead:
```
შენ ხარ senior software architect. გადახედე ამ კოდს/არქიტექტურას:
[კოდი/აღწერა]
შემოგთავაზე: 1) პოტენციური პრობლემები, 2) best practices,
3) refactoring რეკომენდაციები.
```

### 🎓 საგანმანათლებლო როლები

#### სკოლის მასწავლებელი:
```
შენ ხარ გამოცდილი [საგანი] მასწავლებელი. ახსენი [თემა]
[N] წლის მოსწავლეებისთვის. გამოიყენე:
- მარტივი ენა
- ანალოგიები ყოველდღიური ცხოვრებიდან
- ვიზუალური მაგალითები
- შემაჯამებელი quiz 3 კითხვით
```

#### უნივერსიტეტის პროფესორი:
```
როგორც [სფერო] პროფესორი, მოამზადე lecture outline [თემა]-ზე.
სამიზნე: მაგისტრანტები. დრო: 90 წუთი.
შეიცავდეს: თეორიული ჩარჩო, case studies, group discussion topics.
```

### 🎨 კრეატიული როლები

#### Copywriter:
```
შენ ხარ creative copywriter. დაწერე [ad copy/email/social post]
[პროდუქტი/სერვისი]-სთვის.
ტონი: [brand voice]
მიზნობრივი აუდიტორია: [აღწერა]
გამოიყენე copywriting formula: [AIDA/PAS/BAB]
სიგრძე: [N] სიტყვა
```

#### UX Writer:
```
როგორც UX writer, შექმენი microcopy [UI element]-სთვის:
- Clear და concise
- User-friendly ტონი
- Error messages, tooltips, CTAs
- ვარიანტები A/B testing-სთვის
```

## 🔄 Few-Shot Learning ტემპლეიტები

### 📊 კლასიფიკაცია
```
კლასიფიცირე წინადადებები sentiment-ის მიხედვით:

მაგალითი 1:
Input: "ეს პროდუქტი უბრალოდ შესანიშნავია!"
Output: Positive

მაგალითი 2:
Input: "ძალიან იმედგაცრუებული დარჩი."
Output: Negative

მაგალითი 3:
Input: "ჩვეულებრივი პროდუქტია, არაფერი განსაკუთრებული."
Output: Neutral

ახლა კლასიფიცირე:
Input: [ახალი ტექსტი]
Output:
```

### 📝 ფორმატირება
```
გარდაქმენი ბუნებრივი ენა structured JSON-ად:

მაგალითი 1:
Input: "გაასწორე შეხვედრა მარიამთან ხვალ 3 საათზე კაფე ლაგიდზეში"
Output: {
  "action": "create_meeting",
  "person": "მარიამი",
  "time": "15:00",
  "date": "tomorrow",
  "location": "კაფე ლაგიდზე"

მაგალითი 2:
Input: "დამიმახსოვრე რომ ბაბუას დაუგზავნო მილოცვა დაბადების დღეზე"
Output: {
  "action": "create_reminder",
  "person": "ბაბუა",
  "task": "დაბადების დღის მილოცვა",
  "event": "birthday"
}

ახლა გარდაქმენი:
Input: [ახალი ტექსტი]
Output:
```

## 🧠 Chain of Thought (CoT) ტექნიკები

### 🔢 მათემატიკური პრობლემები
```
გადაწყვიტე ეს პრობლემა ნაბიჯ-ნაბიჯ. თითოეულ ნაბიჯში ახსენი რას აკეთებ და რატომ.

პრობლემა: [მათემატიკური ამოცანა]

ნაბიჯი 1: [პრობლემის გააზრება]
ნაბიჯი 2: [მონაცემების იდენტიფიკაცია]
ნაბიჯი 3: [მეთოდის არჩევა]
ნაბიჯი 4: [გამოთვლა]
ნაბიჯი 5: [პასუხის შემოწმება]
საბოლოო პასუხი: [X]
```

### 🧩 ლოგიკური ამოცანები
```
დაასაბუთე შენი პასუხი step-by-step reasoning-ით:

კითხვა: [ლოგიკური ამოცანა]

გააანალიზე:
1. რა ვიცით? (მოცემული ფაქტები)
2. რა არ ვიცით? (უცნობები)
3. რა დაშვებები შეგვიძლია გავაკეთოთ?
4. რა ლოგიკური ნაბიჯებია საჭირო?
5. რა დასკვნამდე მივდივართ?

პასუხი და დასაბუთება: [ახსნა]
```

### 💼 ბიზნეს გადაწყვეტილებები
```
შემოგთავაზე გადაწყვეტილება სისტემატური ანალიზის საფუძველზე:

სიტუაცია: [აღწერა]

ანალიზი:
1️⃣ პრობლემის განსაზღვრა
   [რა არის ძირითადი საკითხი?]

2️⃣ ალტერნატივების იდენტიფიკაცია
   [რა ვარიანტები გვაქვს?]

3️⃣ თითოეული ვარიანტის შეფასება
   Pros: [...]
   Cons: [...]
   Risks: [...]

4️⃣ რეკომენდაცია
   [საუკეთესო გადაწყვეტილება და რატომ]

5️⃣ განხორციელების გეგმა
   [როგორ მოვახდინოთ დანერგვა]
```

```

```

```

## ⚠️ თავიდან ასაცილებელი შეცდომები

### 🚫 პრობლემური Prompt-ები

| შეცდომა | პრობლემა | გამოსწორება |
|---------|----------|-------------|
| "დაწერე რამე" | ზედმეტად ბუნდოვანი | "დაწერე 500 სიტყვიანი ბლოგ პოსტი [თემაზე]..." |
| მრავალი ამოცანა ერთ prompt-ში | კონფუზია | თითოეული ამოცანა ცალკე prompt-ით |
| არასაკმარისი კონტექსტი | ზოგადი პასუხი | დაამატე ყველა რელევანტური დეტალი |
| "გააკეთე უკეთესი" | არაინფორმატიული | "გააუმჯობესე: 1) სტრუქტურა, 2) ტონი, 3) ენა" |
| იგნორირება ფორმატის | არასასურველი output | მიუთითე ზუსტად როგორი ფორმატი გინდა |

### ✅ Prompt Debugging Checklist

შენი prompt არ მუშაობს? შეამოწმე:

- [ ] **Clarity**: არის თუ არა ინსტრუქცია მკაფიო და ცალსახა?
- [ ] **Context**: მეტყველებს თუ არა საკმარისი background ინფორმაცია?
- [ ] **Constraints**: განსაზღვრულია თუ არა შეზღუდვები (სიგრძე, ფორმატი)?
- [ ] **Examples**: საჭიროა თუ არა few-shot examples?
- [ ] **Format**: მიუთითეა თუ არა სასურველი output ფორმატი?
- [ ] **Tone**: დაფიქსირებულია თუ არა ტონი და სტილი?

## 🔧 Advanced Techniques

### 🎯 Self-Consistency
```
გადაწყვიტე ეს პრობლემა 3 განსხვავებული გზით.
თითოეულ მეთოდს დაარქვი სახელი და ახსენი.
ბოლოს შეადარე შედეგები და შემოგთავაზე საუკეთესო პასუხი.

პრობლემა: [ამოცანა]

მეთოდი 1: [სახელი]
[გადაწყვეტა]

## 📊 Prompt შეფასების კრიტერიუმები

### ✅ კარგი Prompt-ის მახასიათებლები

```
┌─────────────────────────────────────────────┐
│ ✓ Specific      - კონკრეტული და ზუსტი     │
│ ✓ Measurable    - გაზომვადი მოთხოვნები     │
│ ✓ Achievable    - რეალისტური              │
│ ✓ Relevant      - რელევანტური კონტექსტი    │
│ ✓ Time-bound    - შეზღუდული (სიგრძე/დრო)   │
└─────────────────────────────────────────────┘
```

### 📈 Prompt-ის ხარისხის შემოწმება

**ცუდი Prompt:**
```
დაწერე რამე მარკეტინგზე
```
**კარგი Prompt:**
```
შენ ხარ digital marketing strategist.
დაწერე 800-სიტყვიანი სტატია "Email Marketing Best Practices"-ზე
B2B SaaS კომპანიებისთვის.
შეიცავდეს:
- 5 actionable tactics
- Real-world examples
- Metrics to track
- Common mistakes to avoid
ტონი: ექსპერტული მაგრამ ხელმისაწვდომი
ფორმატი: შესავალი + დანომრილი სექციები + დასკვნა
```
