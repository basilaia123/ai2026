📌 Make.com-ის ძირითადი ცნებები

Make.com არის ვიზუალური ავტომატიზაციის პლატფორმა, რომელიც აკავშირებს აპლიკაციებსა და API-ებს. იგი მუშაობს შემდეგი პრინციპით:

🔌

Trigger (ტრიგერი)

იწყებს სცენარს. არსებობს Polling (პერიოდული) და Instant (მყისიერი Webhook).

🧱

Module (მოდული)

სცენარის ცალკეული ნაბიჯი (მაგ. Google Sheets, Slack, HTTP).

⚡

Action (მოქმედება)

მოდული, რომელიც ასრულებს ქმედებას (იმეილის გაგზავნა, ბაზაში ჩაწერა).

🔀

Router (როუტერი)

ყოფს ნაკადს რამდენიმე ტოტად ფილტრების პირობების მიხედვით.

🚀 5 მარტივი ავტომატიზაციის სცენარი დამწყებთათვის (ნაბიჯ-ნაბიჯ)

სანამ რთულ ფინტექ პაიპლაინებზე გადავალთ, ააწყვეთ ეს 5 საბაზისო სცენარი Make.com-ზე პრაქტიკული გამოცდილების მისაღებად (მათ შორის OPPA-ს სპეციფიკური ქეისები):

⚡ სცენარი 1 (OPPA-ს ქეისი): Google Sheets (ახალი ტერმინალი) ➔ Telegram შეტყობინება

მიზანი: ცხრილში ახალი ტერმინალის (PayBox) ინსტალაციის მონაცემების დამატებისას, შეტყობინება ავტომატურად გაიგზავნოს OPPA-ს საინსტალაციო გუნდის Telegram არხში.

  1. ტრიგერი (Trigger): აირჩიეთ Google Sheets -> Watch Rows. დაუკავშირეთ Google-ის ანგარიში, აირჩიეთ ტერმინალების აღრიცხვის ფაილი და სამუშაო ფურცელი (Sheet).
  2. მოქმედება (Action): დაამატეთ Telegram Bot -> Send a Text Message or Reply.
  3. კავშირი (Connection): BotFather-ის დახმარებით შექმენით Telegram ბოტი, მიიღეთ Token, ჩასვით Make-ში და ბოტი დაამატეთ თქვენს საინსტალაციო ჯგუფში/არხში ადმინისტრატორად.
  4. Mapping: ტექსტის ველში ჩაწერეთ შეტყობინება და ჩასვით ცვლადები Sheets-იდან (მაგ. დაემატა ახალი ტერმინალის ინსტალაცია: ID - {{1.TerminalID}}, მისამართი - {{1.Address}}).
  5. ტესტი: დააჭირეთ Run once, ჩაწერეთ ახალი ხაზი Sheets-ში (მაგ. TerminalID: 9942, Address: Station Square) და შეამოწმეთ Telegram.

⚡ სცენარი 2 (OPPA-ს ქეისი): Gmail (კლიენტის პრეტენზია) ➔ Slack-ის მხარდაჭერის არხი (#support-alerts)

მიზანი: თუ support@oppa.ge-ზე შემოვიდა მომხმარებლის სასწრაფო იმეილი ხარვეზზე (მაგ. ბანკომატმა თანხა ჩაყლაპა), გუნდმა Slack-ში მყისიერად მიიღოს კრიტიკული შეტყობინება.

  1. ტრიგერი (Trigger): აირჩიეთ Gmail -> Watch Emails. მიუთითეთ Folder: Inbox და Criteria: Simple Filter.
  2. ფილტრი (Filter): Subject-ში ან Body-ში მიუთითეთ სიტყვები: ბანკომატი, თანხა, ჩაყლაპა ან ბალანსი.
  3. მოქმედება (Action): დაამატეთ Slack -> Create a Message. დაუკავშირეთ Slack-ის სამუშაო სივრცე.
  4. Mapping: აირჩიეთ მხარდაჭერის არხი (მაგ. #support-alerts) და ჩაწერეთ ტექსტი Gmail-ის ცვლადების გამოყენებით: 🚨 კრიტიკული ხარვეზი! {{1.sender.emailAddress}} წერს: {{1.subject}}.
  5. ტესტი: დააჭირეთ Run once და გამოუგზავნეთ თქვენს თავს სატესტო იმეილი სათაურით "ბანკომატმა ჩაყლაპა 150 ლარი".

⚡ სცენარი 3 (OPPA-ს ქეისი): Google Drive (დღიური ლოგი) ➔ Claude API (ანომალიების ძებნა) ➔ Gmail Draft (რეპორტი მენეჯმენტს)

მიზანი: Google Drive-ზე ატვირთული ტერმინალების დღიური ლოგების ავტომატური შემოწმება Claude AI-ით, ანომალიების ძებნა და Gmail-ში ყოველკვირეული რეპორტის დრაფტის მომზადება.

  1. ტრიგერი (Trigger): აირჩიეთ Google Drive -> Watch Files in a Folder. მიუთითეთ ლოგების საქაღალდე.
  2. მონაცემის წაკითხვა: დაამატეთ Google Drive -> Download a File ფაილის შინაარსის წასაკითხად.
  3. AI დამუშავება: დაამატეთ Anthropic Claude (ან HTTP მოდული) -> Create a Message. Prompt-ში ჩაწერეთ: "გაანალიზე OPPA-ს ამ ტერმინალების დღიური ლოგი, იპოვე შეცდომები და დაწერე მოკლე შეჯამება მენეჯერისთვის: [ჩასვით ფაილის კონტენტი Download მოდულიდან]".
  4. მოქმედება (Action): დაამატეთ Gmail -> Create a Draft.
  5. Mapping: იმეილის შინაარსში (Body) ჩასვით Claude-ის მიერ გენერირებული ლოგის რეზიუმე.
  6. ტესტი: ატვირთეთ სატესტო ლოგ-ფაილი საქაღალდეში, ჩართეთ Run once და ნახეთ გამზადებული რეპორტის დრაფტი თქვენს Gmail-ში.

⚡ სცენარი 4 (OPPA-ს ქეისი): PayBox ეკრანის ფოტო ➔ Vision AI ➔ Telegram-ის ტექნიკური შეტყობინება

მიზანი: როდესაც PayBox აპარატი აფიქსირებს ხარვეზს და ატვირთავს ეკრანის ფოტოს Google Drive-ზე, AI-მ ამოიცნოს პრობლემა (მაგ. "Paper Jam") და ავტომატურად შეატყობინოს ტექნიკურ ჯგუფს.

  1. ტრიგერი (Trigger): აირჩიეთ Google Drive -> Watch Files in a Folder. მიუთითეთ საქაღალდე, სადაც იტვირთება ტერმინალის სკრინშოტები/ფოტოები.
  2. მოქმედება (Action): დაამატეთ Google AI Studio (Gemini) ან Anthropic Claude -> Generate Content / Create a Message.
  3. მონაცემები: გადაეცით Gemini-ს ატვირთული ფაილის ობიექტი და Prompt: "ამოიცანი ხარვეზი PayBox ტერმინალის ამ სურათზე. დააბრუნე შეცდომის კოდი და მოკლე აღწერა ქართულად.".
  4. შეტყობინება: დაამატეთ Telegram Bot -> Send a Text Message or Reply.
  5. Mapping: ტექსტში ჩაწერეთ: ⚠ ხარვეზი ტერმინალზე! AI ანალიზი: {{2.text}} და დაურთეთ ფოტოს ბმული Google Drive-იდან.

⚡ სცენარი 5 (OPPA-ს ქეისი): საეჭვო ტრანზაქციების აქტივობა ➔ როუტერი ➔ SMS გაფრთხილება მერჩანტს & Slack Alert

მიზანი: OPPA-ს ბილინგიდან მოსული საეჭვო აქტივობის (დიდი ტრანზაქციები მოკლე დროში) გაფილტვრა და მყისიერი შეტყობინება.

  1. ტრიგერი (Trigger): აირჩიეთ Webhooks -> Custom Webhook. დააკავშირეთ OPPA-ს ბილინგის ტრანზაქციების მოდულთან.
  2. ტოტებად დაყოფა: დაამატეთ Router ორ ტოტად დასაყოფად: Normal Event და High Risk Suspicious Event.
  3. ფილტრი: High Risk ტოტზე დააყენეთ ფილტრი: amount > 1000 ან transactions_per_min > 50.
  4. SMS და Slack: ამავე ტოტზე დაამატეთ HTTP მოდული (SMS Office API-ით მერჩანტის მფლობელთან გასაგზავნად) და Slack -> Create a Message (უსაფრთხოების გუნდისთვის).
  5. ტესტი: Postman-იდან გაუშვით სატესტო JSON დიდი თანხით და შეამოწმეთ Slack-ის არხი.

🌐 Webhook ტესტირების მეთოდოლოგია (Webhook.site & Postman)

სანამ სცენარს რეალურ ტრაფიკზე გავუშვებთ, საჭიროა ვებჰუკის სტრუქტურის შემოწმება და ტესტირება:

1. Webhook.site-ის გამოყენება

გამოიყენეთ Webhook.site დროებითი ტესტური URL-ის შესაქმნელად. გაგზავნეთ მასზე cURL მოთხოვნა, რათა დაინახოთ Payload-ის ზუსტი სტრუქტურა და headers-ი.

2. Make.com Webhook Capture

Make.com-ში Custom Webhook-ის შექმნისას დააჭირეთ "Redetermine data structure". გაგზავნეთ სატესტო რექვესთი Postman-იდან. Make ავტომატურად წაიკითხავს ველებს და შექმნის შესაბამის ცვლადებს.

3. Postman — ყველაზე მოსახერხებელი მეთოდი

თუ გაქვთ Postman დაყენებული, გამოიყენეთ შემდეგი კონფიგურაცია:

ეს არის n8n-ის ლოკალურად გასაშვები პროდაქშენ კონფიგურაცია PostgreSQL მონაცემთა ბაზით:

⚠️ უსაფრთხოების გაფრთხილება: პროდაქშენ გარემოში გაშვებამდე აუცილებლად შეცვალეთ ნაგულისხმევი პაროლები და გასაღებები (SecretPassword123, super-secret-key-123) ძლიერი და უსაფრთხო მნიშვნელობებით.
version: '3.8'

services:
  db:
    image: postgres:15-alpine
    container_name: n8n-postgres
    environment:
      POSTGRES_DB: n8n
      POSTGRES_USER: n8n_user
      POSTGRES_PASSWORD: SecretPassword123
    volumes:
      - pg_data:/var/lib/postgresql/data
    restart: always

  n8n:
    image: n8nio/n8n:latest
    container_name: n8n-local
    ports:
      - "5678:5678"
    environment:
      - DB_TYPE=postgresdb
      - DB_POSTGRESDB_DATABASE=n8n
      - DB_POSTGRESDB_HOST=db
      - DB_POSTGRESDB_PORT=5432
      - DB_POSTGRESDB_USER=n8n_user
      - DB_POSTGRESDB_PASSWORD=SecretPassword123
      - N8N_ENCRYPTION_KEY=super-secret-key-123
      - EXECUTIONS_DATA_PRUNE=true
      - EXECUTIONS_DATA_MAX_AGE=168
    volumes:
      - n8n_data:/home/node/.n8n
    depends_on:
      - db
    restart: always
