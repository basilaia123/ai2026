2 საფუძვლები

🧱 Make.com საფუძვლები და HTTP API კონფიგურაცია

📌 Make.com-ის ზოგადი მიმოხილვა და ძირითადი ცნებები

Make.com (ყოფილი Integromat) არის წამყვანი No-code / Low-code ავტომატიზაციისა და ინტეგრაციის პლატფორმა. იგი საშუალებას იძლევა ვიზუალური Drag-and-drop ინტერფეისით დავაკავშიროთ სხვადასხვა აპლიკაციები, მონაცემთა ბაზები და გარე API-ები (მაგალითად, Claude ან Gemini) კოდის წერის გარეშე.

🔑 პლატფორმის ძირითადი ტერმინოლოგია:

  • 💡 სცენარი (Scenario): სამუშაო პროცესი (Workflow), რომელიც აღწერს მონაცემების მოძრაობას და დამუშავებას თავიდან ბოლომდე.
  • 🧱 მოდული (Module): სცენარის ცალკეული შემადგენელი ნაბიჯი (მაგ. Google Sheets, HTTP Module, Slack), რომელიც ასრულებს კონკრეტულ ქმედებას.
  • ⏱ ტრიგერი (Trigger): მოვლენა, რომელიც იწყებს სცენარს. არსებობს Polling (პერიოდული შემოწმება, მაგ. 15 წუთში ერთხელ) და Instant/Webhook (მყისიერი რეაგირება მონაცემების შემოსვლისთანავე).
  • ⚙️ მოქმედება (Action): მოდული, რომელიც ასრულებს დავალებას (მაგ. ელ-ფოსტის გაგზავნა, ახალი სტრიქონის ჩაწერა).
  • 🔀 როუტერი (Router): ყოფს სცენარს რამდენიმე პარალელურ ტოტად პირობების მიხედვით.
  • 🔍 ფილტრი (Filter): ზღუდავს მონაცემთა ნაკადს და ატარებს მხოლოდ იმ მონაცემებს, რომლებიც აკმაყოფილებენ მითითებულ პირობებს.
  • 🔄 იტერატორი (Iterator): ყოფს მასივს (Array) ინდივიდუალურ ელემენტებად, რათა თითოეულზე ცალ-ცალკე შესრულდეს შემდგომი მოდულები.
  • 📊 აგრეგატორი (Aggregator): იტერატორის საპირისპირო მოდული; აერთიანებს რამდენიმე ცალკეულ ოპერაციას/მონაცემს ერთ მასივში (Array).

📊 ავტომატიზაციის პლატფორმების შედარება (Make vs n8n vs Zapier):

მაჩვენებელი Make.com n8n (Self-hosted) Zapier
Compliance (NBG) საშუალო (Cloud) იდეალური (On-Premise) დაბალი (US Servers)
ფასი / Tasks 10k Operations / ~$11 უფასო (ულიმიტო) 750 Tasks / ~$20

📊 ოპერაციები (Operations) და ლიმიტები:

ოპერაცია (Operation) არის Make-ის საზომი ძირითადი ერთეული. ყოველი მოდული, რომელიც აქტიურდება სცენარის გაშვებისას და ასრულებს მოქმედებას, ხარჯავს 1 ოპერაციას.

  • Free Plan-ის ლიმიტი: 1,000 უფასო ოპერაცია თვეში და 100 MB მონაცემთა გადაცემა (Data Transfer).
  • ოპტიმიზაციის წესი: ოპერაციების დასაზოგად გამოიყენეთ Batch Processing (ჯგუფური დამუშავება) Iterator-ის ნაცვლად, სადაც ეს შესაძლებელია.

🚀 6 პოტენციური Make.com / n8n სცენარი OPPA-სთვის

აქ მოცემულია 6 რეალური ბიზნეს-კეისი, სადაც OPPA-ს შეუძლია გამოიყენოს Make.com ან n8n სამუშაო პროცესების ოპტიმიზაციისთვის:

1. PayBox Kiosk ინკასაციის ოპტიმიზაცია (კრიტიკული შევსება)

ტრიგერი: PayBox-ის ბანკნოტის მიმღები (Cash Acceptor) ივსება 90%-ით ➔ Webhook (n8n/Make).

AI/ლოგიკა: სისტემა აჯგუფებს ახლომდებარე აპარატებს, Google Maps API-ით ადგენს ოპტიმალურ სამარშრუტო რუკას.

შედეგი: SMS/Slack შეტყობინება ინკასატორთან და ოპტიმიზებული მარშრუტი, რაც ამცირებს ტრანსპორტირების ხარჯს.

2. ტრანზაქციების ყოველდღიური ავტომატური Reconciliation (ბილინგი)

ტრიგერი: Schedule (ყოველ ღამე 02:00 საათზე) ➔ n8n/Make უკავშირდება Core ბაზას და ბანკების API-ებს.

AI/ლოგიკა: ადარებს OPPA-ს ბილიგის მონაცემებს ბანკის რეალურ ამონაწერებთან.

შედეგი: შეუსაბამო ტრანზაქციები ავტომატურად იგზავნება Accounting-ის Slack-არხში ტიკეტის სახით.

3. PayBox ქსელის მონიტორინგი და აღდგენის სკრიპტი (Auto-Reboot)

ტრიგერი: ტერმინალი გადის ოფლაინში ➔ Make/n8n იღებს მყისიერ Alert-ს ქსელის მონიტორინგის სისტემიდან.

AI/ლოგიკა: API-ით უგზავნის Reboot ბრძანებას GPRS/LTE როუტერს. 5 წუთში კვლავ ამოწმებს პინგს.

შედეგი: თუ კავშირი აღდგა, ტიკეტი იხურება; თუ პრობლემა დარჩა, ავტომატურად იხსნება Jira Service Desk ბარათი.

4. მომხმარებელთა Viber/Telegram ჩატბოტი CRM სინქრონიზაციით

ტრიგერი: კლიენტი Viber/Telegram ბოტიდან აგზავნის შეტყობინებას ჩარიცხვის პრობლემაზე.

AI/ლოგიკა: Claude API იღებს შეტყობინებას, ამოიცნობს ტერმინალის ID-ს, ამოწმებს ბალანსს და ამზადებს პასუხს.

შედეგი: კლიენტი იღებს პასუხს, ხოლო ინციდენტი ავტომატურად იწერება OPPA CRM-ში.

5. ეროვნული ბანკის (NBG) Compliance ანგარიშების გენერატორი

ტრიგერი: Schedule (ყოველთვიურად, თვის 1 რიცხვში) ➔ n8n ლოკალური სერვერიდან იწყებს მუშაობას.

AI/ლოგიკა: აგროვებს მონაცემებს PEP და მაღალი რისკის მერჩანტებზე, აფილტრავს PII მონაცემებს ლოკალურად NBG-ის შაბლონის მიხედვით.

შედეგი: იქმნება დაშიფრული ანგარიში, რომელიც ეგზავნება Compliance ოფიცერს დასამტკიცებლად.

6. PayBox ინციდენტების ავტომატური ბაზა და Notion Knowledge Base Sync

ტრიგერი: PayBox აპარატზე ფიქსირდება ტექნიკური დაზიანება (მაგ. ჩეკის პრინტერის გაჭედვა ან ეკრანის გათიშვა).

AI/ლოგიკა: Claude API იღებს შეცდომის კოდს, ადგენს ნაბიჯ-ნაბიჯ საილუსტრაციო გადაჭრის გზას (Troubleshooting Guide) და აკავშირებს შესაბამისი აპარატის მოდელთან (მაგ. NCR ან Wincor).

შედეგი: Make/n8n ავტომატურად ქმნის ახალ გვერდს **Notion**-ის ინციდენტების დაფაზე (Kanban Board). გვერდის შიგნით ავტომატურად იწერება Claude-ის მიერ გენერირებული ინსტრუქცია ტექნიკოსისთვის.

🧩 გამარტივებული No-Code გზა: Make-ის ნატივი AI მოდულები (Anthropic / OpenAI)

არის თუ არა ეს სესია ზედმეტად რთული არატექნიკური თანამშრომლებისთვის?
ნედლი HTTP API მოთხოვნებისა და JSON სქემების წერა ძირითადად დეველოპერებისთვისაა. ბიზნეს-მენეჯერებსა და ანალიტიკოსებს შეუძლიათ გამოიყენონ Make-ის ნატივი (Native) AI მოდულები, რაც სრულად აქრობს კოდის წერის საჭიროებას:

🛠️ მეთოდი ა: ნატივი Anthropic / OpenAI მოდული

  • უბრალოდ აირჩევთ მოდულს: Anthropic → Create a Message.
  • API გასაღებს ჩაწერთ მარტივ ველში.
  • მოდელს აირჩევთ სიიდან (claude-3-5-sonnet).
  • პრომპტს წერთ ჩვეულებრივ ტექსტად JSON ფორმატის გარეშე.
  • ✓ იდეალურია სწრაფი და მარტივი ამოცანებისთვის.

🌐 მეთოდი ბ: HTTP მოდული (დეველოპერის გზა)

  • წერთ ნედლ POST მოთხოვნას Anthropic-ის API ლინკზე.
  • ხელით აკონფიგურირებთ Headers-სა და JSON სტრუქტურას.
  • ✓ იდეალურია კომპლექსური ფინანსური შემოწმებებისთვის.

🛠️ სხვა ავტომატიზაციის ხელსაწყოები: Zapier, Power Automate და Activepieces

მსხვილ კორპორაციებში Make-ისა და n8n-ის გარდა გამოიყენება სხვა პოპულარული პლატფორმებიც:

💻 Microsoft Power Automate (Desktop & Cloud)

ვისთვისაა: საუკეთესოა გუნდებისთვის, რომლებიც სრულად იყენებენ Microsoft 365-ს (Outlook, Excel, Teams).

🔥 მთავარი უპირატესობა: RPA (Robotic Process Automation) Desktop ვერსია, რომელიც ახდენს ძველი Windows პროგრამების ავტომატიზაციას კლიკების სიმულირებით (რაც API-ის გარეშე მუშაობს).

⚡ Zapier (ყველაზე პოპულარული No-Code პლატფორმა)

ვისთვისაა: აბსოლუტური დამწყებებისთვის, ვისაც სურს 5 წუთში ააწყოს მარტივი ავტომატიზაცია.

🔥 მთავარი უპირატესობა: Zapier Central (AI Bots) — შეგიძლიათ plain English-ით აუხსნათ ბოტს, რა გააკეთოს და AI თავად ააწყობს ინტეგრაციას.

🟢 Activepieces (Open-Source Zapier-ის ლოკალური ალტერნატივა)

ვისთვისაა: კომპანიებისთვის, ვისაც სურს ლოკალურად გაშვებული (Self-hosted) მარტივი სისტემა.

🔥 მთავარი უპირატესობა: აქვს ისეთივე მარტივი ინტერფეისი, როგორიც Zapier-ს, მაგრამ მისი კოდი ღიაა და n8n-ის მსგავსად შეგიძლიათ გაუშვათ OPPA-ს სერვერზე უფასოდ.
