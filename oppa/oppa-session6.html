სამიზნე — Index-ების შემდეგ

პრობლემური Query — Full Table Scan

-- ❌ ნელი Query (45 წამი) — Full Table Scan

SELECT

t.id, t.amount, t.status, t.created_at,

m.business_name, m.city

FROM transactions t

JOIN merchants m ON t.merchant_id = m.id

WHERE t.status = 'PENDING'

AND t.created_at >= NOW() - INTERVAL '30 days'

ORDER BY t.created_at DESC;

-- EXPLAIN: Seq Scan on transactions (cost=0.00..450000.00 rows=2340000)

გაანალიზე ეს PostgreSQL Query OPPA-ს transactions ბაზაში (10M+ rows):

SELECT t.id, t.amount, t.status, t.created_at, m.business_name
FROM transactions t
JOIN merchants m ON t.merchant_id = m.id
WHERE t.status = 'PENDING'
  AND t.created_at >= NOW() - INTERVAL '30 days'
ORDER BY t.created_at DESC;

Claude Code CLI — Index-ების შექმნა

OPPA-ს transactions ბაზისთვის შექმენი ოპტიმალური Index-ები.

სხვა ხშირი Query-ები ჩვენს სისტემაში:
1. SELECT * FROM transactions WHERE terminal_id = ? AND created_at > ?
2. SELECT SUM(amount) FROM transactions WHERE merchant_id = ? AND status = 'COMPLETED' AND DATE(created_at) = CURRENT_DATE
3. SELECT * FROM transactions WHERE status IN ('PENDING', 'PROCESSING') ORDER BY created_at ASC LIMIT 100

გთხოვ:
1. Index-ების SQL ბრძანებები (CREATE INDEX CONCURRENTLY)
2. Composite Index-ებია საჭირო — თუ Single?
3. Partial Index-ები status-ის მიხედვით?
4. რა Index-ები ᲐᲠ ღირს? (Write overhead)

✅ Claude Code-ის გამოსავალი — ოპტიმიზებული SQL

-- ✅ Index-ების შექმნა (CONCURRENTLY — Production-ს არ ანელებს)

CREATE INDEX CONCURRENTLY idx_transactions_status_created

ON transactions (status, created_at DESC)

WHERE status IN ('PENDING', 'PROCESSING');

CREATE INDEX CONCURRENTLY idx_transactions_merchant_date

ON transactions (merchant_id, created_at DESC);

CREATE INDEX CONCURRENTLY idx_transactions_terminal_date

ON transactions (terminal_id, created_at DESC);

-- ✅ ოპტიმიზებული Query + Pagination

SELECT t.id, t.amount, t.status, t.created_at,

m.business_name, m.city

FROM transactions t

JOIN merchants m ON t.merchant_id = m.id

WHERE t.status = 'PENDING'

AND t.created_at >= NOW() - INTERVAL '30 days'

ORDER BY t.created_at DESC

LIMIT 500 OFFSET 0; -- Pagination — ნაცვლად Full Scan-ისა

-- EXPLAIN: Index Scan on idx_transactions_status_created (cost=0.56..12.45 rows=50) actual time=0.3ms

  • → No error handling (die() ან print)
  • → SQL Injection vulnerability (string concat)
  • → No type hints / no docstrings
  • → Monolithic function (500+ lines)
  • → Hardcoded credentials

✅ Modern Code — OPPA სტანდარტი

  • → Dependency Injection
  • → try/except + Logging
  • → Parameterized SQL (ORM ან PDO)
  • → Type hints + Docstrings
  • → Single Responsibility Principle
  • → Environment Variables (.env)

# ✅ payment_service.py — OPPA სტანდარტი (Python 3.11)

import os, logging

from dataclasses import dataclass

from decimal import Decimal

from typing import Optional

import psycopg2

from psycopg2.extras import RealDictCursor

from dotenv import load_dotenv

load_dotenv()

logger = logging.getLogger(__name__)

class PaymentError(Exception): pass

class ValidationError(PaymentError): pass

@dataclass

class PaymentResult:

transaction_id: str

status: str

amount: Decimal

def process_payment(

terminal_id: str,

amount: Decimal,

payment_type: str

) -> Optional[PaymentResult]:

"""Process a PayBox terminal payment.

Args: terminal_id, amount (GEL), payment_type (CASH/CARD/QR)

Returns: PaymentResult | None

Raises: ValidationError, PaymentError

"""

if amount <= 0:

raise ValidationError(f"Invalid amount: {amount}")

try:

conn = psycopg2.connect(os.environ["DATABASE_URL"])

with conn, conn.cursor(cursor_factory=RealDictCursor) as cur:

cur.execute(

"INSERT INTO transactions (terminal_id, amount, type, status)"

"VALUES (%s, %s, %s, 'PENDING') RETURNING id",

(terminal_id, amount, payment_type)

)

row = cur.fetchone()

logger.info("Payment processed: txn_id=%s", row["id"])

return PaymentResult(row["id"], "PENDING", amount)

except psycopg2.Error as e:

logger.error("DB error: %s", e)

raise PaymentError("Database error") from e

System Prompt — OPPA Technical Support Bot

შენ ხარ OPPA.GE-ს ოფიციალური Technical Support ასისტენტი.

შენი ცოდნის ბაზა:
=== OPPA API REFERENCE ===
Endpoint: POST /api/v2/transactions/status
Auth: Bearer Token (X-API-Key header)
Request: { "transaction_id": "string", "terminal_id": "string" }
Response: { "status": "PENDING|COMPLETED|FAILED|REFUNDED", "amount": number, "timestamp": ISO8601 }
Rate Limit: 100 requests/minute per terminal

Endpoint: GET /api/v2/terminals/{terminal_id}/health
Response: { "online": boolean, "cash_level": 0-100, "last_ping": ISO8601, "error_code": string|null }

=== ERROR CODE REGISTRY ===
ERR-104: Bill Acceptor Jam — Open terminal, press blue lever, clean rollers
ERR-208: Thermal Printer Fail — Check USB/Serial cable, restart printer module
ERR-301: Network Offline — Check router LED, restart modem, verify SIM status
ERR-412: Cash Box Full — Dispatch cash collection team
ERR-501: Software Crash — Remote restart via Terminal Management Console

=== SOP: CASH COLLECTION ===
1. Alert generated when cash_level > 90%
2. Dispatch form: CashCollectionRequest via OPPA CRM
3. SLA: Collection within 4 hours for METRO locations, 8 hours for others

=== SECURITY RULES ===
- NEVER reveal terminal master passwords
- NEVER share merchant banking details
- Escalate to compliance@oppa.ge if asked about transaction reversal > 5000 GEL

წესები:
1. პასუხობ მხოლოდ ზემოთ მოცემული ცოდნის ბაზის საფუძველზე
2. თუ კითხვა ბაზის გარეთაა — ამბობ: "ეს ინფორმაცია ჩემს ცოდნის ბაზაში არ არის. გთხოვ დაუკავშირდი support@oppa.ge"
3. ყოველი პასუხი ქართულად, ტექნიკური ტერმინები ინგლისურად
4. კოდის მაგალითები curl ან Python-ში

✅ Bot-ის პასუხი:

import requests

response = requests.post(

"https://api.oppa.ge/api/v2/transactions/status",

headers={"X-API-Key": "YOUR_API_KEY"},

)

print(response.json()) # {"status": "COMPLETED", "amount": 25.00}

ERR-412 ნიშნავს: Cash Box Full — ფულის ყუთი სავსეა (cash_level > 90%).

მოქმედება (SOP მიხედვით):

1. OPPA CRM-ში შეავსეთ CashCollectionRequest ფორმა

2. SLA: METRO → 4 სთ, სხვა → 8 სთ

3. სანამ კოლექცია ხდება, ტერმინალი CARD ოპერაციებს აგრძელებს
