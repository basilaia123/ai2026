📋 Lovable-ის პრომპტი — Terminal Status Monitor

Lovable.dev — სრული App პრომპტი

Build an internal "PayBox Terminal Status Monitor" dashboard for OPPA.GE.

Design:
- Color scheme: Navy (#1B3A5C) and Teal (#2E86AB) theme
- Clean, professional fintech UI
- Georgian language labels

Features:
1. Map/Table View: List of terminals with columns — Terminal ID, Location Name, City, Status (Online/Offline/Error), Last Activity, Cash Level (%)
2. Status Badges: Green (Online), Red (Offline/Error), Yellow (Low Cash < 20%)
3. Filter Bar: Filter by City, Status, Location Type (Metro, Mall, Street)
4. Search: Real-time search by Terminal ID or Location Name
5. "Request Maintenance" Button: Opens a modal to submit a support ticket
6. Summary Cards: Total Terminals | Online Count | Offline Count | Low Cash Count
7. Auto-refresh every 60 seconds

Backend: Connect to a Supabase table called "terminals" with columns: id, location_name, city, status, cash_level, last_activity, location_type

Make it production-ready with loading states and error handling.

Supabase Table Schema — terminals:

CREATE TABLE terminals (

id TEXT PRIMARY KEY, -- T-001, T-002...

location_name TEXT NOT NULL, -- "სადგური სამგორი"

city TEXT DEFAULT 'თბილისი',

status TEXT DEFAULT 'ONLINE', -- ONLINE/OFFLINE/ERROR

cash_level INTEGER DEFAULT 100, -- 0-100 %

last_activity TIMESTAMPTZ,

location_type TEXT -- METRO/MALL/STREET/GAS

);

# ✅ payment_service.py — OPPA სტანდარტი (Python 3.11)

import os, logging

from dataclasses import dataclass

from decimal import Decimal

from typing import Optional

import psycopg2

from psycopg2.extras import RealDictCursor

from dotenv import load_dotenv

load_dotenv()

logger = logging.getLogger(__name__)

class PaymentError(Exception): pass

class ValidationError(PaymentError): pass

@dataclass

class PaymentResult:

transaction_id: str

status: str

amount: Decimal

def process_payment(

terminal_id: str,

amount: Decimal,

payment_type: str

) -> Optional[PaymentResult]:

"""Process a PayBox terminal payment.

Args: terminal_id, amount (GEL), payment_type (CASH/CARD/QR)

Returns: PaymentResult | None

Raises: ValidationError, PaymentError

"""

if amount <= 0:

raise ValidationError(f"Invalid amount: {amount}")

try:

conn = psycopg2.connect(os.environ["DATABASE_URL"])

with conn, conn.cursor(cursor_factory=RealDictCursor) as cur:

cur.execute(

"INSERT INTO transactions (terminal_id, amount, type, status)"

"VALUES (%s, %s, %s, 'PENDING') RETURNING id",

(terminal_id, amount, payment_type)

)

row = cur.fetchone()

logger.info("Payment processed: txn_id=%s", row["id"])

return PaymentResult(row["id"], "PENDING", amount)

except psycopg2.Error as e:

logger.error("DB error: %s", e)

raise PaymentError("Database error") from e

# OPPA Georgian Proofreader Rules
- Ban all corporate barbarisms: "თასქი" -> "დავალება", "ქეისი" -> "შემთხვევა", "დეფოლტად" -> "სტანდარტულად".
