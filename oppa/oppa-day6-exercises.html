სესია 6: გაფართოებული სავარჯიშოები
Lovable, SQL Optimization, Legacy Code და Hermes Agent

15 პრაქტიკული დავალება — შიდა პორტალების აწყობა Lovable-ზე, SQL მოთხოვნების ოპტიმიზაცია, Legacy კოდის მიგრაცია, Hermes Agent-ის კონფიგურაცია და რეალური ბოტის გამართვა.

მოდული I: Lovable Prototyping
🧡 Terminal Status Monitor
1

Terminal Status Monitor-ის დეტალური პრომპტი

დავალება: ჩამოაყალიბე Lovable-ის პრომპტი, რომელიც დაამატებს ტერმინალების რუკაზე "ტექნიკოსების განრიგის მართვის" ფუნქციონალს. ტექნიკოსს უნდა შეეძლოს ხარვეზის სტატუსის შეცვლა (In Progress, Resolved) და კომენტარის დატოვება.

🧡 Expense Approval Flow
2

ხარჯების დამტკიცების აპლიკაციის ლოგიკა

დაწერე პრომპტი Lovable-ისთვის, რათა B2B ხარჯების დამტკიცების პორტალს დაემატოს ავტომატური დაცვის ლოგიკა: 500 ლარზე მეტი ხარჯის მოთხოვნა ავტომატურად გადამისამართდეს ფინანსურ დირექტორთან (CFO), ხოლო 500 ლარამდე ხარჯები — პირდაპირ საოპერაციო მენეჯერთან.

🧡 Supabase Schema
3

Supabase-ის ცხრილების სქემის განსაზღვრა

აღწერე რა ველები (Columns) და მონაცემთა ტიპები (Types) უნდა ჰქონდეს Supabase-ის `paybox_terminals` ცხრილს, რათა Lovable-მა სწორად წაიკითხოს ტერმინალის მისამართი, სტატუსი, ბოლო შეცდომის კოდი და ბოლო განახლების დრო.

მოდული II: SQL & Database Optimization
📊 EXPLAIN ANALYZE
4

Query Planning და ნელი Sequential Scans

OPPA-ს ტრანზაქციების ბაზაზე `EXPLAIN ANALYZE SELECT * FROM transactions WHERE amount > 1000` აჩვენებს ნელ "Seq Scan"-ს. როგორ მოაგვარებ ამ პრობლემას? დაწერე შესაბამისი SQL ინდექსის შექმნის ბრძანება.

📊 Index concurrently
5

უსაფრთხო ინდექსაცია ლაივ ბაზაზე

რატომ არის აუცილებელი OPPA-ს პროდაქშენ გარემოში `CREATE INDEX` ბრძანების ნაცვლად `CREATE INDEX CONCURRENTLY`-ს გამოყენება?

📊 Window Functions
6

Window Functions სატრანზაქციო ანალიზისთვის

დაწერე SQL მოთხოვნა, რომელიც Window Function-ის (მაგალითად, `ROW_NUMBER()` ან `LAG()`) მეშვეობით დაადგენს თითოეული მომხმარებლისთვის წინა ტრანზაქციის თარიღს და გამოთვლის დროს ორ ტრანზაქციას შორის, რათა ვიპოვოთ საეჭვო სწრაფი გადარიცხვები.

მოდული III: Legacy კოდის მოდერნიზაცია
🛠️ Python 2 to 3 migration
7

ძველი Python 2 სკრიპტის მიგრაცია

ჩამოაყალიბე პრომპტი Claude Code-სთვის, რათა მან გადაიყვანოს Python 2-ის ძველი საოპერაციო სკრიპტი Python 3-ზე, ჩაანაცვლოს `print x` ახალი სინტაქსით, გამოასწოროს ძველი string encoding ხარვეზები და გახადოს კოდი UTF-8 ქართული ტექსტისთვის თავსებადი.

🛠️ SQL Injection Fix
8

SQL Injection-ის აღმოფხვრა

გამოსწორე შემდეგი დაუცველი კოდის ნაწყვეტი პარამეტრიზებული SQL მოთხოვნით:

# დაუცველი კოდი:
cursor.execute("SELECT * FROM users WHERE username = '" + user_input + "' AND password = '" + password_input + "'")
🛠️ Pytest generation
9

Pytest ტესტების შექმნა Claude Code-ით

ჩამოაყალიბე პრომპტი, რომელიც დაავალებს AI-ს დაწეროს 3 ერთეული ტესტი (Unit Test) `pytest`-ით OPPA-ს ტრანზაქციების ვალიდაციის ფუნქციისთვის (შეამოწმეთ: დადებითი თანხა, ვალიდური ვალუტა GEL, და არსებული ტერმინალის ID).

Hermes Custom Skill-ის შექმნა AML-ისთვის
