ლექცია 4 — საშინაო დავალება

🎯 Make.com Automations & AI Agents

პრაქტიკული დავალება: Make.com სცენარები, Claude Projects და MCP ინტეგრაცია

⏰ ვადა: ლექციიდან 1 კვირის განმავლობაში | 🎯 ქულა: 10 ქულა (EXPRESS სკალაზე)

🎯 დავალების მიზანი

ამ დავალების მიზანია შეისწავლოთ სამუშაო პროცესების (Workflows) ავტომატიზაცია No-Code პლატფორმაზე, გამართოთ კორპორატიული AI ასისტენტები და დააგეგმოთ უახლესი Model Context Protocol (MCP) არქიტექტურა თქვენი ბიზნეს საჭიროებებისთვის.

📋 დავალების ნაწილები და ქულების განაწილება

  • ნაწილი 1: Make.com პრაქტიკული ავტომატიზაცია 4 ქულა

    ამოცანა: შექმენით და გაუშვით ერთი მუშა სცენარი Make.com-ზე.

    • სცენარის ლოგიკა (მაგალითი):
      • Trigger: ახალი იმეილის მიღება (Gmail-ის მოდული) კონკრეტული ფილტრით.
      • AI Action: იმეილის ტექსტის დამუშავება OpenAI / Anthropic მოდულით, ტონის განსაზღვრა და საპასუხო დრაფტის შედგენა.
      • Output Action: პასუხის ჩაწერა Google Sheets-ში, ან Telegram / Slack არხზე შეტყობინების გაგზავნა.
    • ტესტირება: გაუშვით სცენარი ტესტურ რეჟიმში („Run once“) და დარწმუნდით, რომ ის უშეცდომოდ მუშაობს.
    • დოკუმენტირება: დაურთეთ Make.com-ის სცენარის სქემის სქრინშოტი, სადაც ნათლად ჩანს წარმატებული მუშაობის ნიშნები (მწვანე checkmark-ები) და მიღებული შედეგი (მაგ. ჩაწერილი სტრიქონი Sheets-ში).

  • ნაწილი 2: Claude Projects & System Prompts 3 ქულა

    ამოცანა: გამართეთ გუნდური ან პერსონალური AI ასისტენტი Claude Projects-ში (ან Workspace Agents-ში).

    • ცოდნის ბაზა (Knowledge): ატვირთეთ პროექტში მინიმუმ 2-3 PDF, TXT ან MD ფაილი, რომელიც შეიცავს თქვენი ბიზნესის / სწავლის FAQ-ს ან გაიდლაინებს.
    • სისტემური ინსტრუქცია (Instructions): დაწერეთ დეტალური System Prompt, სადაც მკაფიოდ გაუწერთ აგენტის როლს, ტონს, ცოდნის ფაილებზე დაყრდნობის წესებს და კონფიდენციალურობის შეზღუდვებს (სენსიტიური PII მონაცემების დაცვა).
    • დოკუმენტირება: დაურთეთ დაწერილი System Prompt-ი და პროექტის ინტერფეისის სქრინშოტი.

  • ნაწილი 3: Model Context Protocol (MCP) Blueprint დაგეგმვა 3 ქულა

    ამოცანა: შეადგინეთ კონცეპტუალური Blueprint (არქიტექტურული სქემა და აღწერა), თუ როგორ დააკავშირებდით ლოკალურ AI კლიენტს თქვენს ფაილებთან ან ბაზებთან MCP-ის მეშვეობით.

    • არქიტექტურული დაგეგმვა:
      • რომელ MCP სერვერებს გამოიყენებდით? (მაგ. Filesystem ლოკალური ფაილებისთვის, Postgres/SQLite მონაცემთა ბაზებისთვის, Brave Search ვებ-ძიებისთვის, თუ Puppeteer ბრაუზერის მართვისთვის).
      • აღწერეთ 2 კონკრეტული პრაქტიკული ამოცანა, რომელსაც AI შეასრულებს ამ სერვერების მეშვეობით.
      • როგორ უზრუნველყოფთ მონაცემთა უსაფრთხოებას და კონტროლს? (Tool Approval workflows).
    • დოკუმენტირება: დაწერეთ 150-200 სიტყვიანი არქიტექტურული აღწერა (ან დახაზეთ მარტივი Mermaid/ვიზუალური სქემა).

📊 შეფასების კრიტერიუმები

კომპონენტი ქულა კრიტერიუმები
Make.com ავტომატიზაცია 4 ქულა • მუშა სცენარის აწყობა მინიმუმ 3 მოდულით (OpenAI-ის ჩათვლით) (2 ქ).
• სქემისა და წარმატებული ტესტის სქრინშოტის წარმოდგენა (1.5 ქ).
• სცენარის ლოგიკის მოკლე ახსნა (0.5 ქ).
Claude Projects 3 ქულა • პროექტის შექმნა და დოკუმენტების სწორად ატვირთვა (1 ქ).
• პროფესიონალური System Prompt-ის დაწერა უსაფრთხოების წესებით (2 ქ).
MCP Blueprint 3 ქულა • MCP სერვერების და ამოცანების სწორი შერჩევა (1.5 ქ).
• არქიტექტურული Blueprint-ისა და უსაფრთხოების ლოგიკის გამართული აღწერა (1.5 ქ).
