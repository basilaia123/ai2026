🎯 No-Code Web Application Development

პრაქტიკული დავალება: Lovable.dev პროტოტიპი, Supabase ინტეგრაცია და AI Coding workflows

⏰ ვადა: ლექციიდან 1 კვირის განმავლობაში | 🎯 ქულა: 10 ქულა (EXPRESS სკალაზე)

🎯 დავალების მიზანი

ამ დავალებაში პრაქტიკულად გაივლით სრული ვებ-აპლიკაციის შექმნის ეტაპებს Lovable.dev პლატფორმის გამოყენებით, დააკავშირებთ მას Supabase მონაცემთა ბაზასთან და გააანალიზებთ AI-პირველი IDE ხელსაწყოების (მაგ. Claude Code, Bolt.new, ან Google Antigravity-ს 5-აგენტიანი სისტემა) მუშაობის პრინციპებს.

📋 დავალების ნაწილები და ქულების განაწილება

  • ნაწილი 1: Lovable.dev ვებ-აპლიკაციის პროტოტიპი და გამოქვეყნება 4 ქულა

    ამოცანა: შექმენით და გამოაქვეყნეთ თქვენი პირველი ვებ-აპლიკაციის ინტერფეისი Lovable.dev-ში.

    • აპლიკაციის იდეა: შეარჩიეთ ერთი პრაქტიკული იდეა (მაგ. SaaS Landing Page, Task Tracker, AI Tools Directory, ან პერსონალური ფინანსების პანელი).
    • იტერაციული განვითარება: დააგენერირეთ ინტერფეისი Lovable-ში დეტალური პრომპტების მეშვეობით. გააკეთეთ მინიმუმ 3-4 იტერაციული შესწორება (დიზაინის დახვეწა, ფერების შეცვლა, ღილაკების დამატება).
    • Live Deploy: გამოაქვეყნეთ პროექტი Lovable-ის საჯარო სერვერზე (Live URL).
    • დოკუმენტირება: დაურთეთ აპლიკაციის საჯარო ბმული (Public Live Link) და ინტერფეისის ძირითადი გვერდების სქრინშოტები.

  • ნაწილი 2: Supabase მონაცემთა ბაზის ინტეგრაცია 3 ქულა

    ამოცანა: დააკავშირეთ თქვენი Lovable აპლიკაცია რეალურ მონაცემთა ბაზასთან.

    • კავშირი (1-Click Integration): Lovable dashboard-იდან დააკავშირეთ პროექტი Supabase-თან.
    • ცხრილის გამართვა (Database Tables): შექმენით მინიმუმ ერთი ცხრილი (Table) Supabase-ში მომხმარებლის მიერ შეყვანილი მონაცემების შესანახად (მაგ. დავალებების სია, კონტაქტების ბაზა, ან გამოხმაურებები).
    • მონაცემთა შენახვა: დარწმუნდით, რომ Lovable ინტერფეისის ღილაკებიდან ინფორმაცია რეალურად იწერება Supabase ბაზაში და იტვირთება ეკრანზე.
    • დოკუმენტირება: წარმოადგინეთ Supabase-ის ცხრილის სქრინშოტი, სადაც ნათლად ჩანს მომხმარებლის მიერ შეყვანილი და შენახული მონაცემები.

  • ნაწილი 3: AI Coding workflows (Claude Code / Antigravity IDE) ანალიზი 3 ქულა

    ამოცანა: დაწერეთ 150-200 სიტყვიანი კრიტიკული ანალიზი AI-პირველი IDE ხელსაწყოების შესახებ.

    • ანალიზის საკითხები:
      • რა უპირატესობა აქვს AI-პირველ კოდირების ასისტენტს (Claude Code, Google Antigravity, Bolt.new, ან Replit Agent) ტრადიციულ დეველოპმენტთან შედარებით?
      • რა ტიპის პრომპტები ან ბრძანებები (Commands) არის ყველაზე ეფექტური კოდის შეცდომების გამოსასწორებლად (Debugging)?
      • რა გამოწვევები ან შეზღუდვები დააფიქსირეთ ამ ინსტრუმენტების მუშაობისას?

📊 შეფასების კრიტერიუმები

კომპონენტი ქულა კრიტერიუმები
Lovable.dev პროტოტიპი 4 ქულა • აპლიკაციის გამართული ფუნქციონალი და WOW ეფექტი (2 ქ).
• საჯარო Live ბმულის (Live Link) წარმოდგენა (1 ქ).
• ინტერფეისის გვერდების სრულყოფილი სქრინშოტები (1 ქ).
Supabase ინტეგრაცია 3 ქულა • 1-Click Supabase ინტეგრაციის წარმატებული კავშირი (1 ქ).
• ბაზაში მონაცემთა შენახვა და Supabase ცხრილის სქრინშოტი (2 ქ).
AI Coding workflows ანალიზი 3 ქულა • 150-200 სიტყვიანი გამართული ანალიზი (1.5 ქ).
• IDE ბრძანებებისა და იტერაციული გამართვის პრინციპების სწორი გააზრება (1.5 ქ).
