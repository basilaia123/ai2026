1. რა არის Cursor და რატომ გადავდივართ "Low-Code"-ზე?

Cursor ეს არის VS Code (Visual Studio Code)-ის სრული იდენტური „ტყუპისცალი“ ძრავა, თუმცა მასში ღრმადაა ინტეგრირებული ხელოვნური ინტელექტი. თუ Lovable არის "Black Box" (სადაც თქვენ აძლევთ დავალებას და ის თვითონ აწყობს საიტს, მაგრამ ვერ ხედავთ სრულ პროცესს), Cursor გაძლევთ **აბსოლუტურ კონტროლს** თითოეულ ფაილზე.

🎯 მთავარი უპირატესობა:

თქვენ არ გჭირდებათ იყოთ სენიორ-დეველოპერი იმისათვის, რომ მართოთ პროექტი. Cursor თავად წერს კოდს, აანალიზებს მთელ პროექტს ერთდროულად, წერს ტესტებს და ეძებს შეცდომებს. თქვენი მთავარი როლი არის პროდუქტის მენეჯერობა (Product Manager) და არა უბრალოდ "მწერალი".

ინსტრუმენტი როდის გამოვიყენოთ? კონტროლის დონე აუდიტორია
ChatGPT / Claude ლოგიკის მოფიქრება, კითხვა-პასუხი დაბალი (კოპირება/ჩასმა ხელით) ყველა
Lovable.dev / Bolt.new სწრაფი პროტოტიპი (MVP), ლამაზი UI რამდენიმე წუთში საშუალო (Black Box) არადეველოპერები, დიზაინერები
Cursor AI მასშტაბური პროექტები, ბექენდი (Backend), სრული არქიტექტურა, დაშვება ფაილურ სისტემაზე აბსოლუტური (100% Control) დამწყები & პროფესიონალი დეველოპერები

2. პირველი ნაბიჯები და ინსტალაცია

თუ უკვე იყენებთ VS Code-ს, Cursor-ზე გადასვლა წარმოუდგენლად მარტივია, რადგან ის მხარს უჭერს ყველა არსებულ Extensions-სა და ენას.

1

ჩამოტვირთვა

ეწვიეთ cursor.com-ს და გადმოწერეთ აპლიკაცია თქვენი ოპერაციული სისტემისთვის (Windows, Mac, Linux).

2

VS Code Extensions-ის იმპორტი

ინსტალაციისას, Cursor შემოგთავაზებთ, რომ `VS Code`-დან დააკოპიროს თქვენი საყვარელი Extension-ები (მაგ. Prettier, Python, Live Server, და ა.შ.). დათანხმდით ამ ერთკლიკიან იმპორტს.

3

ანგარიშის (Account) დაკავშირება

Cursor გთავაზობთ უფასო 14-დღიან ულიმიტო Pro ვერსიას ახალი მომხმარებლებისთვის. დააკავშირეთ Google ან GitHub ანგარიში.

💡 "Privacy Mode" ბიზნესებისთვის

თუ კომპანიის კონფიდენციალურ პროექტზე მუშაობთ, Cursor-ის პარამეტრებში (Settings > Privacy) აირჩიეთ `Privacy Mode`. ეს უზრუნველყოფს, რომ თქვენი კოდი არ იქნება გამოყენებული AI მოდელების შემდგომი სწავლებისთვის (Training).

3. მთავარი კლავიატურული კომბინაციები (Cheat Sheet)

Cursor არ არის მხოლოდ Chat ინტერფეისი. მისი მთავარი ძალა კლავიატურით მართვაშია.

4. `@` (Context) - Cursor-ის ყველაზე ძლიერი იარაღი

სტანდარტული ChatGPT-ის პრობლემა ისაა, რომ მას უწევთ გრძელი კოდების ხელით დაკოპირება. Cursor მუშაობს პირდაპირ ფაილურ სისტემაში. თუ გნებავთ AI-მ ზუსტად იცოდეს რაზე საუბრობთ, გამოიყენეთ `@` სიმბოლო Chat-ში ან Composer-ში.

სხვადასხვა ტიპის კონტექსტი:

  • @Files — აკრიფეთ `@index.html`. AI-მ ახლა ზეპირად იცის ამ ფაილის შიგთავსი.
  • @Folders — აკრიფეთ `@components` (საქაღალდე). AI გააანალიზებს ამ საქაღალდეში არსებულ ყველა ფაილს ერთდროულად.
  • @Codebase — აკრიფეთ `@codebase`. Cursor მთლიან, 10,000 ხაზიან პროექტს გადაატარებს ინდექსაციას და შეძლებს იპოვოს კავშირები ფაილებს შორის!
  • @Docs — ეძებთ როგორ გამოიყენოთ რაიმე ახალი ტექნოლოგია (მაგ. Supabase, Tailwind, Stripe)? გამოიყენეთ `@Docs`, აირჩიეთ და Cursor პირდაპირ წაიკითხავს მათ ოფიციალურ დოკუმენტაციას ონლაინ.
  • @Web — AI გადის ინტერნეტში და თავად მოიძიებს პასუხს სტეკოვერფლოუზე (StackOverflow) ან ფორუმებზე თქვენს შეცდომასთან დაკავშირებით.
@auth.py და @database.py ფაილები გავაერთიანოთ. @Codebase-ის გამოყენებით მითხარი კიდევ რომელ ფაილებში დამჭირდება იმპორტების შეცვლა, რომ სისტემა არ დაიმსხვრეს?

5. Composer-თან მუშაობა (Multi-file გენერაცია)

Ctrl + I (ან Cmd + I) იძახებს Composer-ს. ეს არის თქვენი **Personal Senior-Developer**. სანამ Chat მხოლოდ გესაუბრებათ და კოდს გაძლევთ დასაკოპირებლად, Composer **პირდაპირ ქმნის ახალ ფაილებს** და ცვლის არსებულს.

როდის ვიყენებთ Composer-ს?

როდესაც გვჭირდება ისეთი ცვლილება, რომელიც მოიცავს Front-End-საც, Back-End-საც და Database-საც. ან როცა ნულიდან ვაყენებთ პროექტს.

შექმენი ახალი Python პროექტი ონლაინ მარკეტისთვის. მჭირდება შემდეგი სტრუქტურა: - `backend/` ფოლდერი სადაც იქნება FastAPI, - `frontend/` ფოლდერი ლამაზი HTML/CSS-ით, - `requirements.txt` ფაილი. დაიწყე პროდუქტების დამატების მარტივი API-ით.

Composer-ის რეაქცია: ის ერთდროულად შექმნის 3-4 სხვადასხვა ფაილს, გამოიტანს დასტურად (Accept/Reject) და თუ დაადასტურებთ, შეინახავს მათ თქვენსსაკუთარ კომპიუტერზე.

6. .cursorrules - როგორ "გავწვრთნათ" ჩვენი IDE

თუ გსურთ, რომ AI-მ არ გააკეთოს სულელური შეცდომები და დაიცვას თქვენი (ან თქვენი გუნდის) სტანდარტები, შექმენით ფაილი .cursorrules პროექტის მთავარ დირექტორიაში.

Cursor ამ ფაილს წაიკითხავს ყველა პრომპტის წინ. ეს ჰგავს Custom GPT-ის Instructions-ს მთელი პროექტისთვის.

უნივერსალური .cursorrules-ის შაბლონი Front-End-ისთვის:

# You are an expert AI Developer. Follow these rules strictly: ## 1. ტექნოლოგიური სტეკი - გამოიყენე HTML5, Vanilla JavaScript, და Tailwind CSS - არ გამოიყენო jQuery ან სხვა მოძველებული ბიბლიოთეკები - ყველა Icon-ისთვის გამოიყენე FontAwesome ან HeroIcons ## 2. კოდის სტილი - დაწერე კომენტარები ფუნქციების თავზე ქართულ ენაზე, მაგრამ კოდის ცვლადები (Variables) იყავი ინგლისურად. - UI დიზაინი უნდა იყოს Modern, Minimalist, Dark-mode first. - გააკეთე responsive დიზაინი (Mobile-first მიდგომა). ## 3. პრობლემების გადაჭრა (Debugging) - როცა კოდს მისწორებ, არასოდეს წამიშალო სხვა, მუშა კოდი. - ყოველთვის მითხარი ზუსტად რომელ ხაზზე შეცვალე ინსტრუქცია.

გამოიყენეთ საიტი cursor.directory სხვა საუკეთესო .cursorrules ფაილების შესარჩევად და დასაკოპირებლად (მაგ. React-ისთვის, Python-ისთვის, Node.js-ისთვის).

7. Debugging (შეცდომების მოძებნა და გამოსწორება)

Cursor მთლიანად ცვლის შეცდომების გამოსწორების გზას დეველოპმენტში.

  1. ტერმინალის შეცდომები: როცა ეკრანის ქვემოთ Terminal-ში ამოხტება წითელი ერორი (Error), მასთან ჩნდება ლურჯი ღილაკი Add to Chat ან ბრძანება Ctrl+Shift+E. Cursor პირდაპირ წაიკითხავს ერორს, მიხვდება თუ რომელ ფაილშია პრობლემა და იქვე შემოგთავაზებთ შესწორებულ კოდს.
  2. "Fix with AI" (კოდში): თუ კოდში რომელიმე ხაზი შეცდომას შეიცავს, დააჭირეთ ზედვე ღილაკს `Fix with AI`.
  3. Print/Logging დებაგინგი: შეგიძლიათ პრომპტად დაწეროთ: "დამიმატე console.log() ყველა ფუნქციაში @app.js ფაილში, რომ გავიგო სად იჭედება აპლიკაცია".

8. რომელი მოდელი გამოვიყენოთ?

Cursor 2026 წლისთვის მხარს უჭერს ყველა ლიდერ LLM-ს. ზედა მარჯვენა კუთხეში (ან ჩატში) შეგიძლიათ მარტივად შეცვალოთ ისინი:

Claude 3.5 Sonnet / 4.6 - საუკეთესო ყოველდღიური კოდირებისთვის

Anthropic-ის მოდელები ამჟამად უდავო ლიდერები არიან კოდის წერასა და არქიტექტურის გაგებაში. ის არის სწრაფი, ძალიან ზუსტი React/TypeScript-ში და იშვიათად შლის საჭირო კოდებს პროცესისას.

o1-mini (OpenAI reasoning) - საუკეთესო რთული ლოგიკისა და მათემატიკისთვის

როცა Sonnet ვერ უმკლავდება რთულ ალგორითმს, ან როცა ნულიდან ქმნით რთულ ლოგიკას აბაზაზე, გადართეთ o1-mini მოდელზე. ის "იფიქრებს" 10-15 წამი და მოგცემთ უნაკლო, კარგად დაფიქრებულ ამოხსნას.

cursor-small / cursor-fast - სწრაფი რედაქტირებისთვის

კურსორის შიდა, სუპერ სწრაფი მოდელები (რომლებიც მუშაობენ მცირე ლოკალური ცვლილებებისა და Tab Autocomplete-სთვის შეფერხებების გარეშე).

9. სრული პროექტის აწყობის 5-ნაბიჯიანი ციკლი (Cheat Sheet)

ასე გამოიყურება პროფესიონალის ერთი სამუშაო დღე Cursor-ში:

ეტაპი როგორ ვაკეთებთ Cursor-ში ინსტრუმენტი
1. ინიციალიზაცია ვქმნით ცარიელ საქაღალდეს. ვქმნით .cursorrules ფაილს ინსტრუქციებით. Manual Setup
2. კარკასის (Boilerplate) აწყობა ვირჩევთ Composer-ს (Ctrl+I): "ააწყვე React+Tailwind აპლიკაციის ბაზისი 3 გვერდით." Composer
3. ფაილების რეფაქტორინგი ვირჩევთ Ctrl+K ფაილში: "გააუმჯობესე ეს დიზაინი და გახადე Glassmorphism სტილის." Inline Edit
4. ლოგიკის კითხვა/Debugging ვირჩევთ Chat-ს (Ctrl+L) `@backend.py`-ით: "მონაცემთა ბაზიდან მიბრუნებს NULL-ს, რატომ?" Chat + @Context
5. დოკუმენტაციის ინტეგრაცია შემოგვაქვს ახალი ბიბლიოთეკა. ვიყენებთ `@Docs Stripe API`: "დამიმატე კოდში Stripe Payment გადახდის ლოგიკა ამ დოკუმენტაციაზე დაყრდნობით." Chat + @Docs
