📚 ლექციის მიმოხილვა

ლექცია 10-ში ვისწავლეთ, როგორ შევქმნათ მორგებული AI ასისტენტები (Custom GPTs, Claude Projects, Custom Instructions) კონკრეტული ამოცანებისთვის. მორგებული AI ასისტენტები საშუალებას გაძლევთ დაზოგოთ დრო, გაზარდოთ თანმიმდევრულობა და გააუმჯობესოთ შედეგების ხარისხი მუდმივი ინსტრუქციებითა და ცოდნის ბაზით.

🤔 რა არის მორგებული AI ასისტენტი?

მორგებული AI ასისტენტი (Custom AI Assistant) = პერსონალიზებული AI, რომელიც მორგებულია თქვენს სპეციფიკურ ამოცანებზე custom instructions, knowledge base და მუდმივი კონტექსტის მეშვეობით.

რატომ არის საჭირო?

  • დროის დაზოგვა: აღარ გჭირდებათ განმეორებითი ინსტრუქციები ყოველ ჯერზე
  • თანმიმდევრულობა: ერთი და იგივე სტილი და ხარისხი ყოველ შედეგზე

  • ხარისხი: უკეთესი შედეგები, ვიდრე ზოგადი AI-ისგან
  • ეფექტურობა: უფრო სწრაფი სამუშაო პროცესები და ავტომატიზაცია

🎨 ChatGPT Custom GPTs

GPT Components

1. Instructions (ყველაზე მნიშვნელოვანი!)

განსაზღვრავს GPT-ს ქცევას, პერსონას, კომუნიკაციის სტილსა და წესებს.

# Role and Purpose You are a [specific role] that helps [target audience] with [main task]. # Core Responsibilities 1. [Primary function] 2. [Secondary functions] # Communication Style - Tone: [professional/casual/friendly] - Language: [Georgian/English/both] - Format: [structure preference] # Rules and Constraints ALWAYS: - [Rule 1] - [Rule 2] NEVER: - [Constraint 1] - [Constraint 2] # Output Format [Expected structure] # Special Instructions [Additional context]

2. Knowledge Files

  • What: ატვირთეთ დოკუმენტები (PDF, DOCX, TXT, CSV) → GPT მათ რეფერენსად იყენებს
  • Limit: 10 files, 512MB each
  • Use cases: Company policies, style guides, product docs, templates

3. Capabilities

  • ☑️ Web Browsing: მიმდინარე ინფორმაციაზე წვდომა
  • ☑️ DALL-E: სურათების გენერაცია
  • ☑️ Code Interpreter: Python კოდის გაშვება, მონაცემთა ანალიზი

4. Actions (Advanced)

  • Connect to external APIs/services
  • Zapier integration, Google Sheets, custom APIs

როგორ შევქმნათ GPT? (6 ნაბიჯი)

  1. GPT Builder-ში შესვლა: chatgpt.com/gpts/editor
  2. აღწერა: GPT Builder გეხმარებათ ინტერაქტიული დიალოგით
  3. დეტალების კონფიგურაცია: სახელი, აღწერა, ინსტრუქციები, საუბრის დამწყებები
  4. Upload Knowledge: Relevant files (anonymized!)
  5. Enable Capabilities: Web/DALL-E/Code (only if needed)
  6. Publish: Only me / Link / Public

⏱️ დრო:

მარტივი GPT: 5-10 წუთი | რთული GPT ფაილებით: 30-60 წუთი

📝 ეფექტური ინსტრუქციების დაწერა

Instructions = GPT-ს "კონსტიტუცია"

ეს არის ყველაზე მნიშვნელოვანი ნაწილი! კარგი ინსტრუქციები → კარგი GPT.

Best Practices:

  • Be Specific: "დაწერე პროფესიონალური იმეილი" ✗ → "დაწერე ფორმალური ბიზნეს იმეილი ქართულად, პატივისცემის გამომხატველი ტონით" ✓
  • Use Structure: Role, Responsibilities, Style, Rules, Format, Examples
  • ALWAYS/NEVER Sections: მკაფიო საზღვრები და მოლოდინები
  • Include Examples: აჩვენეთ მოსალოდნელი შეტანის/გამოტანის ფორმატი
  • Define Output Format: ზუსტი სტრუქტურა სათაურებით, ბულეტებით და ა.შ.
  • Add Context: Industry specifics, audience, Georgian cultural context

❌ სუსტი ინსტრუქციები:

"შენ ხარ სასარგებლო ასისტენტი, რომელიც წერს კონტენტს."

✅ ძლიერი ინსტრუქციები:

"შენ ხარ სოციალური მედიის კონტენტ მენეჯერი, რომელიც სპეციალიზდება ქართულ ბაზარზე. შენ ეხმარები ბიზნესებს შექმნან ჩამთრევი პოსტები ქართულ და ინგლისურ ენებზე. ყოველთვის იკითხე სამიზნე პლატფორმის შესახებ გენერაციამდე. არასდროს გადააჭარბო პლატფორმის სიმბოლოების ლიმიტს. გამოიყენე ქართული ფორმალური ბიზნეს ტონი. სტრატეგიულად გამოიყენე ჰეშთეგები და ემოჯი."

⚙️ Custom Instructions

Custom Instructions = მუდმივი ინსტრუქციები, რომლებიც ყველა დიალოგზე ვრცელდება.

✅ ხელმისაწვდომია უფასო ანგარიშებზეც!

ChatGPT Custom Instructions (2 Sections)

Section 1: "What would you like ChatGPT to know about you?"

Personal context, background, preferences

  • Industry, Role, Experience
  • Languages (Georgian native, English fluent)
  • Preferences (practical, data-driven)

Section 2: "How would you like ChatGPT to respond?"

Output style, tone, format preferences

  • Tone (professional but conversational)
  • Format (structured, bullet points)
  • ALWAYS/NEVER rules

⚡ სწრაფი Setup:

ChatGPT → Settings → Personalization → Custom Instructions

Claude → Project Settings → Custom Instructions (per project)

🏗️ Agent Architecture - 5 კომპონენტი

1. 🎯 Goal/Task

რა უნდა მიაღწიოს agent-მა (high-level მიზანი).

მაგალითი: "Research top 3 AI startups in Georgia and create structured summary"

2. 🧠 Reasoning Engine (Brain)

LLM, რომელიც ფიქრობს, გეგმავს, გადაწყვეტილებებს იღებს.

როგორ მუშაობს:

  • ანალიზი: რა არის მიმდინარე სიტუაცია?
  • დაგეგმვა: რა ნაბიჯები უნდა გადავდგა?
  • გადაწყვეტილება: რომელი tool გამოვიყენო?
  • Reflection: წარმატებული იყო? რა უნდა შევცვალო?

3. 🛠️ Tools/Actions

Functions რომელიც agent-მა შეუძლია გამოიძახოს.

მაგალითები:

  • Search: Google/web search
  • Code Executor: Python კოდის გაშვება
  • API Calls: External services-თან დაკავშირება
  • File Operations: Read/Write files
  • Calculator: მათემატიკური გამოთვლები
  • Database: SQL queries

4. 💾 Memory

Short-term Memory: მიმდინარე task-ის კონტექსტი და მდგომარეობა

Long-term Memory: წინა sessions-ებიდან ინფორმაცია, სწავლა გამოცდილებიდან

Working Memory: Intermediate results, tool outputs

5. 🔄 Action Loop

Agent-ის მუშაობის ციკლი:

  1. Observe: რა არის მიმდინარე მდგომარეობა?
  2. Think: რა ნაბიჯი უნდა გადავდგა?
  3. Act: შევასრულო მოქმედება (tool გამოყენება)
  4. Observe: რა შედეგი მივიღე?
  5. Reflect: წარმატებული იყო? სჭირდება კორექცია?
  6. Repeat: Until goal achieved

🎨 Agent Design Patterns

1. ReAct (Reasoning + Acting)

როგორ მუშაობს: Agent alternates between reasoning და action.

  • Think: "I need to search for information"
  • Act: Runs search tool
  • Think: "Results show X, now I need Y"
  • Act: Runs another tool

Best for: Research tasks, information gathering

2. Plan-and-Execute

როგორ მუშაობს: Agent first creates complete plan, then executes it.

  • Phase 1: Create detailed step-by-step plan
  • Phase 2: Execute each step sequentially
  • Phase 3: Re-plan if needed based on results

Best for: Complex projects with clear steps

3. Multi-Agent Collaboration

როგორ მუშაობს: Multiple specialized agents working together.

  • Researcher Agent: Gathers information
  • Analyst Agent: Analyzes data
  • Writer Agent: Creates report
  • Reviewer Agent: Quality checks

Best for: Complex projects requiring different expertise

🛠️ პოპულარული Agent Frameworks

1. 🦜 LangChain Agents

  • Platform: Python/JavaScript framework
  • Best For: Custom agents with tools, memory, chains
  • Complexity: Medium (კოდირება საჭიროა)
  • ✅ ძალიან flexible და customizable
  • ✅ უამრავი pre-built tools და integrations
  • ✅ Active community და documentation
  • ❌ Requires Python/JS knowledge

2. 🤖 AutoGPT

  • Platform: Autonomous GPT-4 agent
  • Best For: ავტონომიური complex task execution
  • Complexity: Low (GUI-based setup)
  • ✅ Highly autonomous (minimal user intervention)
  • ✅ Web browsing, file operations, code execution
  • ✅ Self-improving (learns from mistakes)
  • ❌ შეიძლება "გადაუხვიოს" და დახარჯოს ბევრი tokens
  • ❌ Sometimes unstable/unpredictable

3. 👥 CrewAI

  • Platform: Multi-agent collaboration framework
  • Best For: Complex projects with specialized roles
  • Complexity: Medium-High
  • ✅ Multiple agents with different roles/tools
  • ✅ Task delegation between agents
  • ✅ Hierarchical or collaborative workflows
  • ❌ More expensive (multiple LLM calls)

4. 🧪 BabyAGI

  • Platform: Minimal autonomous agent
  • Best For: Simple task list management
  • Complexity: Low-Medium
  • ✅ Simple architecture (easy to understand)
  • ✅ Task prioritization and creation
  • ✅ Lightweight
  • ❌ Limited functionality compared to others

5. 🌐 GPT-4 with Plugins (OpenAI)

  • Platform: ChatGPT native agents
  • Best For: No-code agent-like behavior
  • Complexity: Low (no coding)
  • ✅ Built into ChatGPT Plus
  • ✅ Access to browsing, code interpreter
  • ✅ Plugin ecosystem
  • ❌ Limited autonomy (still user-guided)
  • ❌ Not true "agent" - more like enhanced GPT

⚠️ Agent Limitations & Challenges

1. 💰 Cost Multiplication

Problem: Agents make MANY LLM calls!

მაგალითი: Simple research task:

  • 1. Plan (1 LLM call)
  • 2. Search (1 call)
  • 3. Analyze results (1 call)
  • 4. Search again (1 call)
  • 5. Analyze (1 call)
  • 6. Search third time (1 call)
  • 7. Final analysis (1 call)
  • 8. Create summary (1 call)

Total: 8 LLM calls vs 1 call for regular ChatGPT!

Cost: $0.40-$2.00 per task vs $0.05

2. 🐛 Error Amplification

თუ agent-მა პირველ ნაბიჯზე შეცდომა დაუშვა → ყველა შემდეგი ნაბიჯი არასწორი იქნება!

მაგალითი: Misunderstands task → Searches wrong info → Creates wrong report

3. 🔒 Security Risks

Agent-ს ხელი მიუწვდება tools და APIs → უფრო მაღალი რისკი:

  • Unauthorized API calls
  • Data leakage
  • Unintended actions (delete files, send emails)
  • Prompt injection attacks

Solution: Sandboxing, approval gates, monitoring

4. 🎯 Complexity & Debugging

Agent-ის debug-ი რთულია - არ იცი რომელ ნაბიჯზე შეცდა.

Multi-agent systems კიდევ უფრო რთულია!

✅ Agent Best Practices

1. Start with Narrow Scope

  • ნუ მისცემთ agent-ს ზედმეტად ფართო მიზანს
  • განსაზღვრეთ ცხადი საზღვრები და შეზღუდვები
  • დაიწყეთ მარტივი task-ებით, შემდეგ გააფართოვეთ

2. Human-in-the-Loop

  • Critical decisions უნდა approval ითხოვდეს
  • არ მისცეთ სრული ავტონომია მაღალ-რისკიან ოპერაციებზე
  • Implement checkpoints key stages-ზე

3. Robust Error Handling

  • Implement fallback mechanisms
  • Set maximum iteration limits
  • Monitor costs in real-time
  • Log all actions for debugging

4. Sandboxing & Testing

  • Test in isolated environment first
  • Restrict access to sensitive data/systems
  • Use mock APIs for testing
  • Gradually expand permissions

5. Monitoring & Observability

  • Track all agent actions and decisions
  • Monitor token usage and costs
  • Set up alerts for anomalies
  • Regular performance reviews

✅ Best Practices

1. Start Simple, Iterate

  1. MVP: Create basic assistant with core instructions
  2. Test: Use it for 1 week, collect issues
  3. Improve: Refine instructions based on real usage
  4. Repeat: Continuous improvement

2. Test with Edge Cases

  • 🚩 Ambiguous requests
  • 🚩 Missing information
  • 🚩 Conflicting requirements
  • 🚩 Out-of-scope questions

3. Version Control Instructions

Save instruction versions in a doc:

  • v1.0 (2025-01-15): Initial version
  • v1.1 (2025-01-22): Added tone guidelines
  • v1.2 (2025-02-01): Fixed formatting issues

4. Document Your Setup

  • Purpose: What does it do?
  • Use Cases: When to use it?
  • Input Format: How to prompt it?
  • Knowledge Base: What files are included?
  • Limitations: What it can't do?

5. Security & Privacy

  • 🔴 Don't upload: Confidential data, customer PII, passwords
  • 🟡 Public GPTs: Anyone can see instructions & files!
  • 🟢 Private GPTs: Safe for sensitive knowledge
  • 📋 Enterprise: Required for company data

🔧 Troubleshooting

Problem Solution
GPT არ მიჰყვება ინსტრუქციებს Use "ALWAYS"/"NEVER", make more explicit, add examples
Inconsistent outputs Define exact output template, include structure example
Knowledge files არ გამოიყენება Reference files in instructions, simplify filenames, split large files
Too slow / expensive Disable unused capabilities, reduce file sizes, shorten instructions
Too generic responses Add more context, upload domain-specific files, be more specific
