📚 სასწავლო გიდის შესახებ

ეს სასწავლო გიდი დაგეხმარებათ:

  • მოემზადოთ ლექციისთვის და გაიმეოროთ მასალა
  • დაეუფლოთ ავტომატიზაციის პლატფორმებს (Zapier, Make.com, Power Automate)
  • შეისწავლოთ No-Code ინსტრუმენტები (Bubble, Webflow, Airtable)
  • შექმნათ რეალური workflow-ები თქვენი სამუშაოსთვის
  • გაიაზროთ AI-ის ინტეგრაცია ავტომატიზაციაში

რეკომენდებული დრო: 3-4 საათი თეორია + 4-6 საათი პრაქტიკა

🎯 სწავლის მიზნები

ლექციის დასრულების შემდეგ უნდა შეგეძლოთ:

ცოდნა (Knowledge):

  • განმარტოთ, რა არის ტრიგერი, მოქმედება, Workflow და ინტეგრაცია
  • ჩამოთვალოთ ავტომატიზაციის 5 ძირითადი სარგებელი
  • აღწეროთ განსხვავება Zapier-სა და Make.com-ს შორის
  • იცოდეთ, როდის რომელი პლატფორმა გამოიყენოთ
  • გაიგოთ No-Code პლატფორმების შესაძლებლობები

უნარები (Skills):

  • შექმნათ მარტივი, 2-3 ნაბიჯიანი Zap-ი Zapier-ში
  • ააწყოთ ვიზუალური სცენარი Make.com-ზე (5+ მოდულით)
  • გამოიყენოთ ფილტრები და როუტერები workflow-ებში
  • მოახდინოთ AI-ის (ChatGPT API) ინტეგრაცია ავტომატიზაციაში
  • შეადაროთ სამი პლატფორმა და აირჩიოთ შესაბამისი

გამოყენება (Application):

  • დაზოგოთ 5-10 საათი კვირაში ავტომატიზაციით
  • შექმნათ მინიმუმ 3 პრაქტიკული workflow თქვენი სამუშაოსთვის
  • გამოიყენოთ No-Code ინსტრუმენტები MVP-ის შესაქმნელად
  • ააწყოთ AI-ზე დაფუძნებული ავტომატიზაციის პროცესი
  • გადაწყვიტოთ "შექმნა vs. ყიდვა" ტიპის პრობლემები ავტომატიზაციაში
🔑 ძირითადი კონცეფციები და ტერმინები

ტრიგერი (Trigger)

მოვლენა, რომელიც ავტომატიზაციას იწყებს. მაგალითად: ახალი იმეილის მიღება, ფორმის შევსება, ფაილის ატვირთვა.

მოქმედება (Action)

ის, რაც ტრიგერის შემდეგ უნდა მოხდეს. მაგალითად: შეტყობინების გაგზავნა, ფაილის შენახვა, მონაცემთა ბაზაში ჩანაწერის შექმნა.

სამუშაო პროცესი / სცენარი (Workflow / Scenario)

ტრიგერებისა და მოქმედებების სრული, ლოგიკური თანმიმდევრობა, რომელიც ასრულებს კონკრეტულ ამოცანას.

ინტეგრაცია (Integration)

სხვადასხვა აპლიკაციისა და სერვისის ერთმანეთთან დაკავშირების პროცესი მონაცემების გასაცვლელად.

როუტერი (Router - Make.com-ში)

მოდული, რომელიც საშუალებას გაძლევთ, workflow დაანაწილოთ რამდენიმე განშტოებად და სხვადასხვა პირობის მიხედვით სხვადასხვა მოქმედება შეასრულოთ.

იტერატორი (Iterator - Make.com-ში)

მოდული, რომელიც ამუშავებს მონაცემთა მასივს (Array) და თითოეული ელემენტისთვის ცალ-ცალკე ასრულებს მოქმედებებს.

მონაცემთა საცავი (Data Store - Make.com-ში)

Make.com-ის შიდა, მარტივი მონაცემთა ბაზა, რომელიც საშუალებას გაძლევთ შეინახოთ და გამოიყენოთ ინფორმაცია სცენარებს შორის.

🛠️ ინსტრუმენტები და პლატფორმები, რომლებიც უნდა იცოდეთ
  • Zapier: ყველაზე პოპულარული და მარტივი პლატფორმა 6000-ზე მეტი ინტეგრაციით. იდეალურია სწრაფი და მარტივი ავტომატიზაციისთვის.
  • Make.com (Integromat): მძლავრი და ვიზუალური პლატფორმა კომპლექსური workflow-ების შესაქმნელად. გამოირჩევა მოქნილობითა და უფრო ხელსაყრელი საფასო პოლიტიკით.
  • Microsoft Power Automate: Microsoft-ის ეკოსისტემაში ღრმად ინტეგრირებული ინსტრუმენტი, რომელიც RPA (რობოტული პროცესების ავტომატიზაცია) შესაძლებლობებსაც მოიცავს.
  • Bubble: No-Code პლატფორმა სრულფასოვანი ვებ-აპლიკაციების (SaaS, მარკეტპლეისები) შესაქმნელად კოდის გარეშე.
  • Webflow: პროფესიონალური, დიზაინზე ორიენტირებული ვებსაიტების შესაქმნელი No-Code პლატფორმა.
  • Airtable: მონაცემთა ბაზისა და ცხრილების ჰიბრიდი, რომელიც იდეალურია CRM-ების, კონტენტის კალენდრებისა და სხვადასხვა შიდა სისტემების ასაწყობად.
💪 პრაქტიკული უნარები, რომლებსაც უნდა დაეუფლოთ
❓ მიმოხილვის კითხვები
  1. აღწერეთ სიტუაცია, როდესაც Make.com-ს აირჩევდით Zapier-ის ნაცვლად და დაასაბუთეთ თქვენი არჩევანი სამი არგუმენტით.
  2. რა არის "იტერატორის" (Iterator) მთავარი ფუნქცია Make.com-ში? მოიყვანეთ პრაქტიკული მაგალითი, სადაც მისი გამოყენება აუცილებელია.
  3. როგორ უკავშირდება AI ავტომატიზაციას? აღწერეთ ერთი მაგალითი, სადაც AI "აჭკვიანებს" ტრადიციულ ავტომატიზაციას.
  4. თქვენ გჭირდებათ შიდა ხელსაწყოს შექმნა კომპანიისთვის, რომელიც თანამშრომლების შვებულებებს აღრიცხავს. რომელ No-Code პლატფორმას გამოიყენებდით (Bubble vs Airtable) და რატომ?
  5. რა არის მთავარი უსაფრთხოების რისკი, რომელიც გასათვალისწინებელია ავტომატიზაციის პლატფორმების გამოყენებისას და როგორ შეამცირებდით ამ რისკს?

📖 სწავლის გეგმა

კვირა 1: ავტომატიზაციის საფუძვლები

დღე 1: ავტომატიზაციის კონცეფციები და Zapier (2 საათი)

შესასწავლი მასალა:

  • რა არის ავტომატიზაცია და რატომ არის მნიშვნელოვანი
  • ტრიგერის, მოქმედების, Workflow-ს კონცეფციები
  • Zapier-ის ძირითადი ფუნქციები და შესაძლებლობები

აქტივობა:

  • დარეგისტრირდით Zapier-ზე (უფასო ანგარიში)
  • შექმენით პირველი მარტივი Zap (Gmail → Slack ან Google Sheets → Email)
  • დატესტეთ workflow და დააკვირდით, როგორ მუშაობს

შემოწმება:

  • მუშაობს თუ არა თქვენი პირველი Zap?
  • გესმით, რა არის ტრიგერი და მოქმედება?

დღე 2: Zapier-ის გაღრმავება (2.5 საათი)

შესასწავლი მასალა:

  • Multi-step Zaps და Paths
  • ფილტრების გამოყენება (პირობითი ლოგიკა)
  • Formatter და Delay ფუნქციები
  • ფასების პოლიტიკა და შეზღუდვები

აქტივობა:

  • შექმენით 3-ნაბიჯიანი Zap ფილტრით
  • მაგალითი: Gmail (label: "Urgent") → Filter → Todoist + Slack
  • დაამატეთ Formatter მონაცემების ფორმატირებისთვის

შემოწმება:

  • შეგიძლიათ ფილტრის გამოყენება?
  • მუშაობს თუ არა მრავალსაფეხურიანი Zap-ი?

დღე 3: Make.com - ვიზუალური ავტომატიზაცია (2.5 საათი)

შესასწავლი მასალა:

  • Make.com-ის პლატფორმის მიმოხილვა
  • Scenario Builder და ვიზუალური ინტერფეისი
  • განსხვავება Zapier-სა და Make.com-ს შორის
  • როუტერები და იტერატორები

აქტივობა:

  • დარეგისტრირდით Make.com-ზე (უფასო 1000 ოპერაცია/თვეში)
  • შექმენით პირველი სცენარი ვიზუალურ კონსტრუქტორში
  • გამოიყენეთ როუტერი სხვადასხვა გზებისთვის
  • მაგალითი: RSS Feed → Router → Twitter + LinkedIn + Facebook

შემოწმება:

  • გესმით Make.com-ის ვიზუალური ინტერფეისი?
  • შეგიძლიათ როუტერის გამოყენება?

დღე 4: Make.com - Advanced ფუნქციები (2 საათი)

შესასწავლი მასალა:

  • იტერატორები და მასივების დამუშავება
  • Data Stores-ის გამოყენება
  • Error Handlers
  • Webhooks და real-time ინტეგრაცია

აქტივობა:

  • შექმენით სცენარი იტერატორით (მაგ. Google Sheets-ის რიგები → ცალ-ცალკე დამუშავება)
  • დაამატეთ Error Handler მოდული
  • დატესტეთ, როგორ მუშაობს შეცდომების დამუშავება

შემოწმება:

  • იცით, როგორ გამოიყენოთ იტერატორი?
  • გაქვთ Error handling-ი თქვენს სცენარში?

კვირა 2: No-Code და AI ინტეგრაცია

დღე 5: Microsoft Power Automate (1.5 საათი)

შესასწავლი მასალა:

  • Power Automate და Microsoft 365 ეკოსისტემა
  • Cloud Flows vs Desktop Flows (RPA)
  • AI Builder და დადასტურების workflow-ები

აქტივობა:

  • თუ გაქვთ Microsoft 365 ანგარიში, შექმენით მარტივი Flow
  • მაგალითი: OneDrive-ში ახალი ფაილი → დადასტურების workflow
  • ან უბრალოდ გაეცანით დემო ვიდეოებს

შემოწმება:

  • იცით, როდის გამოიყენოთ Power Automate?
  • გესმით, რა არის RPA?

დღე 6: No-Code პლატფორმები (2 საათი)

შესასწავლი მასალა:

  • Bubble - ვებ-აპლიკაციები კოდის გარეშე
  • Webflow - პროფესიონალური ვებსაიტები
  • Airtable - მონაცემთა ბაზა და კოლაბორაცია

აქტივობა:

  • დარეგისტრირდით Airtable-ზე
  • შექმენით მარტივი მონაცემთა ბაზა (მაგ. Project Tracker ან CRM)
  • დაამატეთ ხედები (Grid, Kanban, Calendar)
  • შექმენით ფორმები მონაცემთა შეგროვებისთვის

შემოწმება:

  • შეგიძლიათ Airtable-ის ბაზის შექმნა?
  • იცით, რომელი პლატფორმა რისთვის არის საუკეთესო?

დღე 7: AI ინტეგრაცია (2.5 საათი)

შესასწავლი მასალა:

  • ChatGPT API ინტეგრაცია Zapier/Make.com-ში
  • AI-ზე დაფუძნებული მომხმარებელთა მხარდაჭერის ავტომატიზაცია
  • დოკუმენტების დამუშავება AI-ით (OCR + ChatGPT)
  • კონტენტის გენერირების პროცესი

აქტივობა:

  • შექმენით AI-ზე დაფუძნებული workflow (ChatGPT + Zapier ან Make.com)
  • მაგალითი 1: იმეილი → ChatGPT (სენტიმენტის ანალიზი) → გუნდთან გადამისამართება
  • მაგალითი 2: Google Sheets-ის იდეები → ChatGPT (კონტენტის გენერირება) → სოციალურ მედიაში გამოქვეყნება

შემოწმება:

  • შეგიძლიათ AI-ის ინტეგრაცია workflow-ში?
  • მუშაობს თუ არა AI-ზე დაფუძნებული ავტომატიზაცია?

დღე 8: პრაქტიკული პროექტი (3 საათი)

შესასწავლი მასალა:
